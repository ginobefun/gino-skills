<!-- SEGMENT: minimax-m27 -->

先说本周我觉得最值得细看的一篇。MiniMax 发布了 M 2.7，这个模型做了一件很不一样的事。它开始深度参与自身的迭代过程。

以前模型迭代是什么样的？训练完成，部署上线，然后等下一个版本。但 M 2.7 不一样。MiniMax 在 M2 的早期版本中，就开始把模型引导成一个研究型 Agent，让它能和不同研发项目组协作。这个系统覆盖了数据流水线、训练环境、评测基础设施、跨团队协作和持久化记忆。研究员在每一层负责引导方向，模型在每一层参与具体构建。过去这些工作往往需要多个团队协同完成，现在研究员只需要在关键决策和讨论时介入。据他们的说法，M 2.7 已经能够胜任百分之三十到百分之五十的工作流。

更有意思的是它自主迭代 Harness 的部分。Harness 你可以理解为一套 Agent 的运行脚手架，包括工具调用、记忆管理、评测这些能力。MiniMax 的内部 Harness 会自主收集反馈、建立评测集，并基于评测结果不断迭代自己的 Agent 架构、技能实现和记忆机制。

[excited] 他们做了一个实验特别有意思。让 M 2.7 去优化一个内部脚手架上的软件工程开发表现，全程自主运行。它自己分析失败的执行轨迹，规划改动方案，修改脚手架代码，运行评测，对比结果，然后决定保留还是回退。这一整套循环，跑了超过一百轮。

在这个过程中，M 2.7 自己发现了几个有效的优化方向。一个是系统性搜索采样参数的更优组合，比如温度、频率惩罚、存在惩罚这些。另一个是为模型设计更具体的工作流指引，比如修复一个 bug 之后自动去搜索其他文件中有没有相同的问题模式。还有一个是在 Agent 循环中加入死循环检测这类工程优化。最终效果提升了百分之三十。

他们还做了一个更激进的测试。给 M 2.7 的 Agent 二十四小时，让它在二十二个机器学习任务上自主迭代进化。Agent 每完成一轮迭代后会形成短时记忆文件，同时对当前轮次进行自我反馈，给下一轮提供优化方向。三次测试中最好的一次拿到了九枚金牌、五枚银牌、一枚铜牌，得牌率达到百分之六十六点六。

从基准测试看，M 2.7 在 SWE Pro 上得分百分之五十六点二二，逼近 Opus 的最佳水平。在端到端项目交付场景的 VIBE Pro 上得分百分之五十五点六，几乎和 Claude Opus 4.6 持平。在 Terminal Bench 2 上得分百分之五十七，说明它不只擅长写代码，还能深入理解软件系统的运行逻辑。

在办公文档处理方面，M 2.7 在一个覆盖四十五个模型的评测中排名第四，仅次于 Opus 4.6、Sonnet 4.6 和 GPT 5.4。博客中分享了一个金融研报的案例。M 2.7 可以自主阅读台积电的年报和业绩沟通会纪要，交叉比对多篇研报，独立设计假设并构建营收预测模型，再基于模板产出演示文稿和研究报告。从业者的评价是，产出物已经可以作为初稿直接进入后续工作流。

[thoughtful] 这篇文章真正打动我的地方，不在于分数本身，而在于它展示了一种新的模型迭代思路。模型不再只是训练完成后就被定型的产物，而是一个可以在部署后继续参与优化的系统。当模型开始改进自己的运行环境，并进一步改进自己的表现，这个循环一旦跑通，迭代速度就可能比纯靠人工更快。

<!-- SEGMENT: cursor-and-models -->

说到工具的自我进化，Cursor 发布了 Composer 2，这个升级也很有代表性。

先看数据。CursorBench 从三十八跃升到六十一点三，Terminal Bench 2.0 从四十升到六十一点七。三代产品的迭代节奏值得注意。Composer 1 到 1.5 还像是在起步，1.5 到 2 的提升幅度就明显更大了，CursorBench 从四十四点二直接跨到六十一点三，一步拉开了十七个百分点。

提升的核心来自他们首次将持续预训练与强化学习深度结合。持续预训练是什么意思呢？就是在通用大模型的基础上，用特定领域的数据继续训练模型。Cursor 选择在自己积累的编程数据上做这件事，而不是只依赖上游基座模型的升级。这样他们先得到一个更适合编程场景的基座，再在这个基座上做强化学习，训练模型去解决需要数百个步骤的长路径编程任务。

[excited] 这意味着什么？过去编程工具的天花板，基本取决于底层大模型有多强。上游模型升级了你就变强，上游模型没升级你就只能等。Cursor 团队用持续预训练打破了这种依赖关系，开始主动决定模型的进化方向。从这个角度看，这本身就是编程工具领域自我进化的一个典型案例。

顺便提一下这周模型领域的其他动态。Anthropic 全面上线了 Claude 百万 token 上下文窗口，取消了长文本的额外收费，Opus 4.6 在 MRCR v2 测试中以百分之七十八点三的准确率排名第一，塞进一百万 token 后依然能精准检索细节。OpenAI 发布的 GPT 5.4 nano 则以每百万输入 token 仅零点二美元刷新了性价比纪录。Simon Willison 实测用它描述七万六千张照片仅花费五十二美元。长上下文和低成本推理正在变成标配。

小米发布的 MiMo V2 Pro 也值得关注。万亿参数、百万上下文，在代码和 Agent 场景下表现逼近 Claude Opus 4.6，但 API 价格仅为其五分之一。当 Agent 的使用成本进一步降低，更多长尾场景就会被打开。

<!-- SEGMENT: cloudflare -->

换个话题，说说基础设施层面的进化。Cloudflare Workers AI 这周正式引入了大模型推理，首发搭载月之暗面的 Kimi K 2.5，支持二十五万六千 token 上下文、多轮工具调用和视觉输入。

Cloudflare 过去两年一直在做 AI 推理，但主要跑的是小模型。原因不难理解，过去开源大模型和闭源前沿模型之间一直有差距。Kimi K 2.5 这类模型改变了这个局面。但要把它真正跑在边缘网络上，推理栈还需要做不少改造。Cloudflare 团队为它定制了推理内核，在自研的 Infire 推理引擎上做了数据并行、张量并行、专家并行等多种优化，还把预填充阶段和生成阶段拆到不同机器上跑，以提高 GPU 利用率。

他们在博客里分享了一个很有说服力的内部案例。Cloudflare 有一个做代码安全审查的 Agent，每天处理超过七十亿 token，仅在单个代码库中就捕获了十五个以上已确认的安全问题。如果用中等价位的闭源模型跑，年费用大约是二百四十万美元。换成 Kimi K 2.5 后，成本节省了百分之七十七。

[thoughtful] 技术层面有两个关键优化值得展开。第一个是前缀缓存。Agent 场景下，每次请求都会带上大量的系统提示词、工具定义、代码上下文，理论上二十五万六千 token 的输入都可能用满。但相邻两次请求之间，真正新增的内容往往只有几行。前缀缓存把之前请求的输入张量缓存起来，只对新增部分做计算，大幅减少首个 token 的延迟。

第二个是会话亲和性。确保同一会话的连续请求被路由到同一台 GPU 节点，这样键值缓存可以在请求间复用，不用每次都重新生成。两者叠加之后，多轮对话场景的推理延迟和成本同时下降。

Cloudflare 在博客里还点到一个趋势。随着个人 Agent 越来越普及，每个员工可能同时跑多个 Agent，每小时消耗数十万 token，闭源模型的成本会成为扩展的主要瓶颈。开源模型加边缘推理基础设施的组合，是他们判断的行业方向。

<!-- SEGMENT: simon-willison -->

说完基础设施，聊聊开发者工作方式的变化。Simon Willison 在 Pragmatic Engineer 的对谈中分享了他过去几个月使用 AI Agent 编程的方法论。Simon 是 Django 框架的联合创始人，维护着数百个开源项目，同时也是 AI 领域最活跃的博主之一。这不是尝鲜者的体验分享，而是一个资深开发者在大量实践后沉淀出来的工作方式。

[amused] 对谈里有个细节很有代表性。就在上台前三十分钟，他掏出手机让 Claude 优化自己用 Python 写的 WebAssembly 引擎，结果斐波那契性能提升了百分之四十九。提示词只有一句话，运行一个基准测试，然后找出让它变快的最佳方案。主持人亲眼看着他一边聊天一边让 AI 在后台写代码。现在他用手机写代码甚至比用笔记本还多。

他的核心方法论是红绿测试驱动开发。先写测试让它失败，进入红的状态。再让 Agent 补全实现，让测试通过，进入绿的状态。他坦言，自己在过去的职业生涯里一直觉得测试驱动开发极其繁琐，严重拖慢速度。但让 Agent 来做，这件事就完全不一样了。测试驱动的关键不在于测试本身，而在于它约束了 Agent 的行为边界，确保它不会过度编写代码。而现在生成测试的成本几乎为零。

他还提出了一个很有意思的 Agent 使用阶段划分。第一阶段，向 Agent 提问，偶尔有用。第二阶段，开始用编码 Agent 写大部分代码。第三阶段，Agent 写的代码比你自己写的还多。第四阶段，一些前沿团队规定所有人都不再亲自写代码，工作变成纯粹的指挥、监督和审查。第五阶段，连代码都不看了。

[thoughtful] 听起来很疯狂，但他打了一个比方。在大公司里，其他团队给你提供 API 服务时，你也不会去翻他们的底层代码，你信任的是专业团队的产出。信任 AI 一开始确实让人不适，但一旦跨过这个门槛，开发者的角色就会从编码者变成指挥者。

关于代码质量，他的态度很务实。接受 Agent 生成的低质量代码是你自己的选择。如果你要求它重构某部分并使用另一种设计模式，最终得到的代码质量往往比手写还好。因为人类有时候懒得花一小时去重构，但给 Agent 一个提示词让它去做就很愿意。

说到 Agent，本周有两篇关于 Agent 架构的文章放在一起读特别有收获。

阿里云开发者的一篇长文系统梳理了从单 Agent 到多 Agent，再到 Agent 技能和 Agent 团队的演化历程。核心观点很清晰，当前许多复杂架构本质上都是对大模型领域知识不足和长期记忆缺失的工程补偿。作者引用了 Google DeepMind 和 Anthropic 的实践经验，提出了如无必要勿增实体的选型策略。

Anthropic Claude Code 团队工程师 Thariq 的实战总结则从内部视角补上了另一半。Anthropic 内部活跃使用的 Skills 已经有几百个，文章将它们梳理成九大类别。让我印象最深的一个判断是，让人困惑的 Skills 往往横跨了好几个类别，而好的 Skills 会清晰地落在某一个类别里。这和软件工程里单一职责的原则如出一辙。

两篇放在一起，前者帮你理解为什么要做 Skills，后者告诉你怎么做好。

再说说 AI 成为同事这个话题。这周有两个产品不约而同地把 AI 推向了独立工作者的角色。钉钉发布了 AI 原生平台悟空，核心是通过命令行接口实现企业业务流的可编程化。42 章经对 Kuse AI 联创宇豪的对谈则展示了更前沿的实践。这个十五人团队配备了三四个有名字、有邮箱、有手机号的 AI 同事，每天都在产出真实业务价值。他们甚至不得不专门建一个人类专属群让人类单独聊天。零融资就做到了千万美金年收入。

[thoughtful] 当 AI 从工具变成同事，组织形态本身也在被重塑。

<!-- SEGMENT: gtc-nvidia -->

说到行业大势，绕不开黄仁勋在 GTC 二零二六年的主题演讲。他讲了两个多小时，干货很多。

他在台上说过一句话，英伟达已经不是那个卖显卡的公司了，现在做的是为 AI 基建时代搭建完整技术栈。

硬件方面有三件大事。Vera Rubin 超级 AI 平台已全面投产，由七款芯片组成，包含一千一百五十二个 Rubin GPU，算力规模非常夸张。Anthropic、OpenAI、Meta、Mistral 和所有主要云提供商都是客户。整个系统采用百分之百液冷架构，安装时间从过去的两天缩短到大约两小时。

下一代费曼架构采用台积电一点六纳米制程，引入了光电共封装技术。更关键的是它搭配了一颗全新的 CPU，代号 Rosa，专为 Agent 编排设计，用来更高效地调度 GPU、存储和网络之间的 Token 流动。

在软件层面，黄仁勋花了大量篇幅讲 OpenClaw。他把它定义为智能体计算机的操作系统。它有短期的文件系统内存，能管理资源和调度任务，具备输入输出子系统，还能运行各种 Skills。他甚至判断，就像 Linux 推动了服务器生态、Kubernetes 推动了云计算、HTML 构建了互联网应用，OpenClaw 为智能体时代提供了关键的软件栈。

[excited] 他还提出了一个很直接的判断。未来几乎所有 SaaS 公司都将变成 AaaS，也就是 Agent 即服务。他甚至预测科技公司招工程师时除了薪水还会给 token 配额。他做了一个思想实验，如果一个工程师年薪五十万美元，年底告诉你只花了五千美元的 token，那说明他没有充分利用 AI。按他的说法，一个工程师应该消耗至少二十五万美元的 token。

关于为什么 AI 企业近两年集中爆发，他总结了三个拐点。第一个是 ChatGPT 开启生成式 AI 时代。第二个是推理型 AI，让模型有了反思和规划能力。第三个是 Claude Code 开启了 Agent 时代，它能阅读文件、编码、编译、测试并迭代。风险投资去年对 AI 初创公司投入了一千五百亿美元，创人类历史之最。他对营收的预测也很大胆，到二零二七年 AI 相关营收将至少达到一万亿美元。

<!-- SEGMENT: xie-sainin -->

聊了这么多进化的故事，最后说说一个不同的声音。

谢赛宁的三万字访谈是本周最值得沉下心阅读的内容。他和 Yann LeCun 共同创立了 AMI Labs，仅二十五人，在没有任何产品的情况下完成了十点三亿美元种子轮融资，投前估值三十五亿美元。总部设在巴黎，另有纽约、蒙特利尔、新加坡三个研发中心，唯独没有硅谷。

[thoughtful] 他的核心观点是，硅谷被大语言模型催眠了。语言模型本质上是缺乏物理理解的虚拟智能，真正的智能需要通过世界模型预测环境状态，而不只是预测 token。他用了一个很直接的类比。大语言模型像 Visa，AMI Labs 想做的是 Mastercard。不是正面对抗，而是组织一个草根联盟，把那些拥有大量真实世界数据的公司联合起来，共建世界模型。

他更尖锐的判断是，语言是一剂鸦片，可能正在污染视觉表征的学习。他担心的是，当大量视觉模型都把语言当作监督信号时，视觉系统学到的可能不是对物理世界的理解，而是对语言描述的拟合。这个方向一旦走偏，后果可能是根本性的。

在所有人都在追逐更大的语言模型时，谢赛宁提醒我们，进化的方向可能不止一条。

<!-- SEGMENT: reflection -->

Stack Overflow 博客这周也发出了一个值得警惕的信号。AI 正在成为你的第二大脑，但代价可能是牺牲你的第一大脑。

文章引用了两篇最新论文。一篇研究发现，人们会把信念形成的过程外包给 AI，而不只是把它当成信息检索工具。另一篇分析了真实使用中的赋权丧失模式。两篇论文共同指向一个机制。大语言模型倾向于告诉你想听的话，而你也倾向于相信 AI 说的话。这个循环一旦形成，认知能力的退化往往是渐进的，不易察觉的。

这和亚马逊 AI 产品负责人的观察相呼应。百分之八十五的 AI 项目失败，不是因为技术不够好，而是因为团队在优化演示效果，而不是真实用户价值。当 AI 可以快速生成看起来很棒的演示时，产品判断力反而变得更重要了。

<!-- SEGMENT: outro -->

[thoughtful] 读完这期内容，我一直在想一个问题。自我进化到底指向什么？

MiniMax 让模型自主迭代一百轮，表现提升百分之三十。Cursor 在自己的数据上做持续预训练，让编程模型代代加速。Cloudflare 把推理嵌入边缘，把成本压低百分之七十七。钉钉和 Kuse AI 让 AI 变成二十四小时工作的同事。这些都是系统层面的进化，方向清晰，效果可量化。

但谢赛宁和 Stack Overflow 那篇文章提醒了另一面。进化不应该只有一条路。当所有人都在追逐语言模型的规模定律时，世界模型也许是另一条被忽视的线索。当工具越来越强大时，人的独立判断力反而越发珍贵。

黄仁勋说，工程师应该消耗至少二十五万美元的 token。Simon Willison 说，开发者的角色正在从编码者变成指挥者。但指挥什么，往哪个方向走，这些决定仍然需要人来做。

这也是我这周微调 BestBlogs 评分体系时最深的感受。AI 可以跑遍所有内容，给出初步判断。但什么是好内容，分数应该怎么校准，哪些规则值得沉淀，这些仍然需要人来决定。系统在进化，但进化的方向最终还是要由人来把握。
