# 视频视觉设计指南

## 整体定位

BestBlogs 早报视频是播客音频的**视觉增强版**，音频是主体，画面与音频内容同步切换。视频的作用是让观众在有屏幕时获得更丰富的信息接收体验。

**核心原则**:
- **画面丰富**: 每 8-15 秒切换一张 slide，避免长时间停留在同一画面
- **音视频同步**: slides 内容与音频播讲内容一一对应
- **素材优先**: 优先使用原文配图、截帧、GIF，不足时再生成
- **简洁克制**: 画面服务于内容，不喧宾夺主

## 品牌视觉体系

延续 BestBlogs 已有的品牌设计语言（与 daily-digest 杂志风 HTML 和海报一致）:

### 配色

| 角色 | 色值 | RGB | 用途 |
|------|------|-----|------|
| 墨蓝 | `#1a365d` | 26, 54, 93 | 主色、标题、品牌 |
| 琥珀 | `#d97706` | 217, 119, 6 | Top 1 唯一强调色 |
| 米白 | `#fefdfb` | 254, 253, 251 | 背景 |
| 深炭灰 | `#374151` | 55, 65, 81 | 正文文字 |
| 中灰 | `#6b7280` | 107, 114, 128 | 辅助信息 |
| 淡灰分隔 | `#e5e7eb` | 229, 231, 235 | 分隔线 |

### 字体

| 用途 | 字体栈 | 特点 |
|------|--------|------|
| 标题/品牌 | Georgia, 'Noto Serif SC', serif | 衬线体，书卷气质 |
| 正文/摘要 | -apple-system, 'PingFang SC', 'Noto Sans SC', sans-serif | 无衬线，清晰易读 |
| 数字/评分 | 'Inter', 'SF Mono', monospace | 等宽对齐 |

### 视觉气质

高端设计杂志（Monocle、Kinfolk）+ 瑞士国际主义排版:
- 大量留白，信息有呼吸感
- 无渐变、无发光、无粒子效果
- 平面设计，不堆叠阴影
- 克制的动效，以缓入缓出、淡入淡出为主

## 视频结构（场景分解）

### 画面尺寸

- **竖版**: 1080×1920 (9:16)，适合手机/短视频平台
- **横版**: 1920×1080 (16:9)，适合 YouTube/B站
- 默认生成**横版**（16:9），竖版作为可选参数

### 场景流程

```
[Scene 1] 品牌开场         3-5秒
[Scene 2] 今日关键词        3-4秒
[Scene 3] Top 1 精讲       对应音频时长（2-3分钟）
[Scene 4] Top 2 精讲       对应音频时长
[Scene 5] Top 3 精讲       对应音频时长
[Scene 6] 速览引导         2-3秒
[Scene 7] 速览卡片组       对应音频时长（3-4分钟）
[Scene 8] 品牌结尾         3-5秒
```

## 各场景设计

### Scene 1: 品牌开场（3-5 秒）

```
┌──────────────────────────────────┐
│                                  │
│                                  │
│                                  │
│        BestBlogs 早报            │  ← 墨蓝衬线体，居中
│                                  │
│        2026年3月8日               │  ← 中灰，小号
│                                  │
│                                  │
│                                  │
└──────────────────────────────────┘
背景: 米白 #fefdfb
```

**动效**: 标题从 opacity 0 → 1 淡入（0.5s ease-out），日期延迟 0.3s 淡入

### Scene 2: 今日关键词（3-4 秒）

```
┌──────────────────────────────────┐
│                                  │
│                                  │
│     ┌─────┐ ┌─────────┐ ┌────┐  │
│     │ AI  │ │ Claude 4│ │ MCP│  │  ← 墨蓝细边框胶囊标签
│     └─────┘ └─────────┘ └────┘  │
│                                  │
│                                  │
└──────────────────────────────────┘
```

**动效**: 标签依次淡入，间隔 0.2s

### Scene 3-5: 精讲场景（Top 1/2/3）— Slides 模型

精讲场景是视频的核心。每条精讲 **2.5-3 分钟**，由 **8-12 张 slides** 组成，每张 **8-15 秒**，slides 间用 0.8s 交叉淡入淡出切换。

#### 6 种 Slide 类型

| 类型 | 布局 | 用途 | 背景 |
|------|------|------|------|
| `cover` | 全幅大图 + 底部暗色渐变 + 标题/来源叠加 | 引入文章，建立第一印象 | 图片 / 米白 |
| `source-card` | 白色卡片 UI（模拟文章预览/推文嵌入） | 展示原始来源信息 | 极淡蓝 |
| `problem` | 居中大字文本 | 提出核心问题，吸引注意力 | 极淡蓝 |
| `point` | 左图(60%) + 右文(40%)，或纯文本 + 琥珀色左竖线 | 展示观点 + 配图 | 米白 |
| `quote` | 居中斜体衬线 + 装饰性引号（琥珀色 60% 透明） | 金句引用，增加权威感 | 极淡蓝 |
| `takeaway` | 暗色墨蓝遮罩图片 + 白色文字，或居中文本 | 个人思考/总结 | 图片叠加 / 米白 |

#### 标准 Slides 编排（以 150 秒精讲为例）

```
[cover]        ~15s  全幅封面图 + 标题 + 来源（淡入）
[source-card]  ~10s  来源预览卡片（含评分徽章）
[problem]      ~12s  居中问题文字（"为什么这很重要？"）
[point]        ~20s  观点 1 + 文内图片 1（左图右文）
[point]        ~15s  观点 1 补充 + 另一张图（切换图片增加视觉变化）
[point]        ~20s  观点 2 + 文内图片 2
[point]        ~18s  观点 3 + 文内图片 3
[quote]        ~10s  金句引用（斜体居中）
[takeaway]     ~15s  个人思考（暗色图片叠加）
[takeaway]     ~15s  总结要点（淡出过渡到下一精讲）
```

#### Top 1 特殊处理

- Cover slide 序号使用琥珀色
- 整体视觉稍强调（更大字号）

#### Ken Burns 效果

所有含图片的 slide 应用缓慢放大（1.0 → 1.05），保持静态画面的视觉动感。

#### 无图片时的 Fallback

- `cover`/`point`/`takeaway` 无图时退化为纯色背景 + 居中大字排版
- `problem`/`quote` 本身就是纯文字设计，不受影响
- 自动 fallback：若未提供 `slides` 数据，系统从 `points`/`quote`/`summary` 自动生成等分 slides

### Scene 6: 速览引导（2-3 秒）

```
┌──────────────────────────────────┐
│                                  │
│                                  │
│      ── 快速速览 ──              │  ← 墨蓝，衬线体
│                                  │

│                                  │

```

每条速览内容显示为一张简洁卡片，随音频节奏切换:

```
┌──────────────────────────────────┐
│                                  │
│  ❹                               │
│  Google 发布 Gemini 2.5 Flash    │  ← 墨蓝衬线体标题
│  API 更新                        │
│                                  │
│  新增多模态输入和更长上下文窗口，  │  ← 深炭灰正文
│  价格降低 30%                     │
│                                  │
│  Google Developers Blog | 92     │  ← 中灰辅助行
│                                  │
└──────────────────────────────────┘
```

**切换方式**: 卡片间用淡入淡出切换（0.4s），每张卡片显示时长对应音频段落。

## 动效规范

### 全局原则

- **淡入为主**: 所有元素出现使用 `opacity: 0 → 1`，配合轻微上移（`translateY: 10px → 0`）
- **缓动函数**: `ease-out` 或 `cubic-bezier(0.25, 0.46, 0.45, 0.94)`
- **时长控制**: 0.4-0.8s，不超过 1s
- **无花哨转场**: 不使用翻页、旋转、缩放、弹性等效果

### 文字动效

| 元素 | 动效 | 时长 |
|------|------|------|
| 标题 | 淡入 + 上移 10px | 0.5s |
| 正文行 | 逐行淡入，间隔 0.15s | 0.4s |
| 辅助信息 | 纯淡入 | 0.3s |
| 关键词标签 | 依次淡入，间隔 0.2s | 0.3s |

### 图片动效

| 场景 | 动效 | 时长 |
|------|------|------|
| 图片出现 | 淡入 | 0.8s |
| 图片切换 | 交叉淡入淡出 | 0.8s |
| Ken Burns（可选） | 缓慢放大 1.0→1.05 | 全段时长 |

Ken Burns 效果（缓慢放大/平移）可选，用于长段落图片保持静态画面的视觉动感。

## 图片处理

### 尺寸与裁剪

- 所有图片统一处理为 16:9 比例（横版）或 9:16（竖版）
- 使用 `object-fit: cover` 裁剪，不变形
- 建议渲染分辨率: 图片区域 1152×648 (16:9 视频中)

### 质量要求

- 最小宽度: 800px（低于此分辨率不使用，改用 fallback）
- 格式: JPG/PNG/WebP
- 如需缩放，使用 Lanczos 算法保持清晰度

### 内容匹配要求（音视频同步关键）

图片必须与当前播讲内容的上下文**精确匹配**:
- 每个 slide 的图片/文字必须对应音频正在讲述的段落
- 讲"背景介绍"时显示 cover slide，讲"观点 1"时显示 point slide + 相关图片
- 不使用与内容无关的装饰性图片（通用 banner、stock photo、广告图）
- 视频类内容优先使用原视频关键帧截图，配合 "视频中演示了……" 的讲解
- 播客类内容如无合适图片，使用生成图补充（保持 BestBlogs 品牌风格）

**素材来源优先级**: 原文封面 > 文内配图/GIF > 视频截帧 > 推文媒体 > AI 生成图

### AI 生成图风格规范

当原文无合适图片时，使用 `image-gen` 生成，所有生成图必须保持统一风格:
- **风格**: 极简科技插画（Minimalist tech illustration）、扁平设计、干净线条
- **配色**: 与 BestBlogs 品牌一致——墨蓝 `#1a365d`、米白 `#fefdfb`、灰色调
- **禁止**: 写实照片风格、卡通/漫画风格、鲜艳配色、包含文字
- **同一期节目的所有生成图必须风格一致**，不能出现混搭

### 叠加文字时的处理

当图片上需要叠加文字:
- 添加半透明渐变遮罩: `linear-gradient(transparent 30%, rgba(0,0,0,0.6) 100%)`
- 文字使用白色 `#ffffff`，确保对比度
- 或使用图片下方区域放置文字（不覆盖图片）

## Remotion 组件架构

### 组件层级

```
<BestBlogsPodcast>              # 根组件
  ├── <BrandIntro />            # Scene 1: 品牌开场
  ├── <KeywordsScene />         # Scene 2: 关键词
  ├── <DeepDiveScene />         # Scene 3-5: 精讲（×3）— Slides 模型
  │     ├── <SlideTransitionWrapper />  # 交叉淡入淡出
  │     └── <SlideRenderer />          # 按 slide.type 分发
  │           ├── <CoverSlide />       # 全幅封面
  │           ├── <SourceCardSlide />   # 来源预览卡片
  │           ├── <ProblemSlide />      # 居中问题
  │           ├── <PointSlide />       # 左图右文
  │           ├── <QuoteSlide />       # 金句引用
  │           └── <TakeawaySlide />    # 思考总结
  ├── <QuickReviewIntro />      # Scene 6: 速览引导
  ├── <QuickCard />             # Scene 7: 速览卡片（×7）
  └── <BrandOutro />            # Scene 8: 品牌结尾
```

### 关键 Props

```typescript
// 根组件 props = video-data.json 的内容
interface BestBlogsPodcastProps {
  date: string;
  keywords: string[];
  brandName: string;
  brandSlogan: string;
  audioFile: string;
  totalDuration: number;
  items: VideoItem[];  // 精讲项含 slides[] 数组
}
```

### 帧率与时间

- FPS: 30
- 1 秒 = 30 帧
- 使用 `useCurrentFrame()` 和 `interpolate()` 控制动画
- 音频同步: 通过 `<Audio>` 组件加载 podcast.mp3，Remotion 自动同步
- Slides 切换: 使用 `TransitionSeries` + `fade()` 实现交叉淡入淡出（0.8s）

---

## 设计禁忌

- **不要** 使用蓝紫渐变科技感
- **不要** 添加发光、粒子、毛玻璃效果
- **不要** 给卡片加投影
- **不要** 使用超过两种强调色
- **不要** 画面塞满信息，留白是设计的一部分
- **不要** 文字动画过快或过花哨
- **不要** 在速览区使用和精讲一样复杂的布局
- **不要** 图片和文字争夺视觉焦点，同一时刻只有一个主角
