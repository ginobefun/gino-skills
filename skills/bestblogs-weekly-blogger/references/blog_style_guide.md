## 语言与排版

### 基本规则

- 中文为主，英文专有名词保留原文（Claude Code, GPT-5.4, Vibe Coding 等）
- 中英文之间加空格：`模型 Claude Sonnet 4.6 的升级`
- 中文和数字之间加空格：`本期精选 20 篇内容`
- 标点使用中文标点（句号、逗号、顿号），英文词汇内部用英文标点

### 排版禁忌

- 减少引号的使用，尤其不要在每个概念首次出现时都加引号
- 减少破折号（——）的使用，改用逗号或分句
- 避免复杂的长句和从句嵌套，一句话表达一个意思
- 不要用 `>` 引用块来展示文章摘要，直接融入叙述
- 不要用列表（bullet points）罗列文章，用段落自然串联

## 叙事风格

### 核心定位

博客是一篇有主题、有观点的深度导读，不是周刊的搬运或摘要合集。读起来应该像在和一个熟悉 AI 领域的朋友聊天，他这周读了很多好东西，正在和你分享最值得关注的内容。

### 语气

- 自然、流畅、不端着
- 有信息密度但不堆砌
- 有个人判断但不武断
- 偶尔幽默但不刻意

### 好的叙述示例

> Claude Sonnet 4.6 这次升级幅度不小。百万 token 上下文窗口、全面升级的编码和 Agent 能力，实测 59% 的场景优于上代旗舰 Opus 4.5，但价格和 Sonnet 4.5 持平。

而不是：

> **Claude Sonnet 4.6** 是 Anthropic 最新发布的模型。它具有以下特点：（1）百万 token 上下文窗口；（2）编码和 Agent 能力升级；（3）价格与上代持平。

前者是叙述，后者是笔记。博客应该是前者。

### 过渡与连接

章节之间需要过渡句，让读者感受到逻辑的递进而非话题的跳转：

> 模型在变强，工具在进化，但最终落到每个人手上，体验是什么样的？

## 篇幅与层次感

这是博客质量的关键。每篇文章的展开程度应该不同，形成明显的层次感。

### 重点文章（3-5 篇，每篇 200-400 字）

- 本周最重磅的发布或趋势
- 用户特别喜欢或推荐的内容
- 深度方法论或框架性文章
- 有争议或有深层思考的内容

重点展开时要包含：
- 文章的核心论点或关键发现
- 从原文中提取的具体细节（数据、案例、关键引用）
- 个人解读或与其他内容的关联

### 普通文章（10-15 篇，每篇 50-100 字）

- 简要提及核心价值
- 自然嵌入链接
- 可以 2-3 篇串联在同一段落中

### 一笔带过（2-5 篇，每篇 1-2 句）

- 在章节末尾简要提及
- 与主题关联但不需展开

## 文章引用与链接

### 链接格式

统一使用 BestBlogs 站内链接（`readUrl` 字段），格式为自然嵌入的 Markdown 链接：

```markdown
Boris Cherny 在[两场深度访谈](https://www.bestblogs.dev/article/xxx)中分享了 Claude Code 的设计哲学。
```

而不是：

```markdown
Boris Cherny 分享了 Claude Code 的设计哲学。
- 链接: https://www.bestblogs.dev/article/xxx
```

### 引用原文观点

转述文章观点时保持准确，但用自己的语言重新组织：

> 他有一个判断我很认同：编程问题被解决之后，下一个前沿是让 AI 从执行者变成主动提出想法的同事。

## 作者归属

这是信息准确性的关键，也是之前容易出错的地方。

### 区分平台与作者

很多文章发布在知名平台上，但实际作者不是平台创始人。必须通过原文内容确认实际作者：

- martinfowler.com 上的文章 → 可能是 Kief Morris、Rahul Garg 等 ThoughtWorks 同事撰写
- Lenny's Podcast 上的访谈 → 嘉宾是主角，Lenny 是主持人
- 某公司官方博客 → 查看 byline 确认具体作者

### 播客/视频的特殊处理

- 翻译或搬运的内容要注明原始来源和翻译方
- 例如：`Insight Partners 联合创始人 Jerry Murdock 的访谈，由跨国串门儿计划翻译为中文`
- 不要写成 `跨国串门儿计划的播客`，这会混淆内容来源

## 图片使用

### 图片规范

每个主要章节前插入一张配图。写作阶段使用裸文件名作为占位符，阶段六生成图片后会替换为实际 URL：

```markdown
![章节描述](bestblogs-issue-{N}-{seq}.png)
```

### 图片数量与位置

- 通常 6-8 张图
- 首图放在开头段之后（标题图）
- 每个主要章节标题后放一张
- 尾图固定：「保持好奇，我们下周见」

### alt 文本

alt 文本简洁描述图片内容，与章节主题相关：

```markdown
![模型越来越强，角色也在跟着变](...)
![记忆正在成为核心](...)
```

## 开头与结尾

### 开头（2-3 段）

1. 第一段：点明本期关键词和主题，概述本周最值得关注的趋势
2. 第二段：个人近期动态（1-2 句），自然过渡到正文
3. 首图

开头不要过长，3-5 句话即可切入正题。避免套话式的问候。

### 结尾

1. 回扣主题关键词，用 3-5 句话收束全文
2. 个人感想，点到为止，不要说教
3. 链接到 BestBlogs.dev 完整周刊：`完整的 N 篇精选文章可以在 [BestBlogs.dev](https://www.bestblogs.dev/newsletter/issueN) 上查看。`
4. 固定结语：`保持好奇，我们下周见。`
5. 尾图

## 常见问题与反模式

### 避免：均匀铺开

每篇文章都用差不多的篇幅介绍 → 读起来像流水账。应该有明显的详略之分。

### 避免：按分类机械组织

MODELS 一节、DEV 一节、PRODUCT 一节 → 按主题线索组织，一个章节可以跨分类。

### 避免：摘要堆砌

把每篇文章的 summary 复制粘贴过来加个过渡句 → 用自己的理解重新组织信息。

### 避免：过度使用格式标记

不要在每个文章标题上加粗、每个关键概念加引号、每段用列表 → 用段落自然叙述。

### 避免：信息不准确

不核实就写「Martin Fowler 认为...」→ 查看原文确认实际作者后再写。

### 避免：AI 写作痕迹

- 不要用「值得关注的是」「不容忽视」「令人印象深刻」等套话
- 不要每段都用「此外」「另外」「同时」开头
- 不要用「总的来说」「综上所述」做总结
- 不要用过多的排比句和三段式结构
