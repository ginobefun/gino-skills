# 各平台内容规格参考

## 推文 (X/Twitter)

| 维度 | 规格 |
|------|------|
| 单条字符限制 | 280 字符（中文约 140 字） |
| 推文串上限 | 25 条（建议 3-8 条） |
| 图片 | 最多 4 张，支持 PNG/JPG/GIF |
| 视频 | 最长 2 分 20 秒 |
| 链接 | 自动缩短为 23 字符 |

### 推文串最佳实践
- 第 1 条: Hook，核心观点或反直觉结论
- 每条聚焦一个子观点，独立可读
- 最后一条: 总结 + CTA
- 编号格式: 不需要显式编号，Twitter 自动串联

## 微信公众号

| 维度 | 规格 |
|------|------|
| 标题 | 最长 64 字符 |
| 正文 | 无硬性限制，建议 1500-4000 字 |
| 摘要 | 最长 120 字符 |
| 封面图 | 推荐 900×383 px（2.35:1） |
| 文内图 | 宽度 ≤ 1080px |

### 排版规范
- 段落短: 每段 2-3 句，适配手机屏
- 小标题多: 每 300-500 字一个小标题
- 加粗多: 关键信息加粗标注
- 引用块: 用于金句或数据引用
- 图文比: 每 800-1000 字配一张图

## 小红书

| 维度 | 规格 |
|------|------|
| 标题 | 最长 20 字，含 emoji |
| 正文 | 最长 1000 字，建议 300-800 字 |
| 标签 | 最多 10 个，建议 5-8 个 |
| 图片 | 1-9 张，推荐竖版 3:4 |

### 内容风格
- 列表化: 用 emoji 分点
- 口语化: 像朋友分享，不学术
- 实用性: 每篇有明确的 takeaway
- 标签策略: 混合热门标签 + 精准标签

## 即刻

| 维度 | 规格 |
|------|------|
| 正文 | 建议 100-500 字 |
| 图片 | 最多 9 张 |
| 链接 | 可嵌入 |

### 内容风格
- 社区讨论感: 像发朋友圈
- 观点优先: 先说结论，再补论据
- 互动引导: 结尾抛问题邀请讨论

## 知乎

| 维度 | 规格 |
|------|------|
| 标题 | 建议问答格式或观点式 |
| 正文 | 建议 2000-5000 字 |
| 图片 | 无限制 |

### 内容风格
- 论证严密: 有数据支撑
- 引用规范: 注明出处
- 结构清晰: 大纲式小标题
- 可用 Q&A 格式展开复杂话题

## 博客 (ginonotes.com)

| 维度 | 规格 |
|------|------|
| 格式 | Markdown + YAML frontmatter |
| 字数 | 1500-3000 字 |
| 封面图 | 推荐 1200×630 px |
| 标签 | 3-5 个 |

### Frontmatter 模板
```yaml
---
title: "标题"
date: YYYY-MM-DD
tags: [tag1, tag2, tag3]
description: "SEO 描述，150 字以内"
cover: "/images/cover.jpg"
---
```
