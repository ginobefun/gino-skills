# 播客脚本风格指南

## 核心理念

这不是一份填空模板，而是一组创作原则。每期播客的结构应该由内容本身决定，不是由模板框住。

**一句话定位**: 一个懂技术的朋友在和你聊他最近看到的有意思的东西。

**风格锚点**: 脚本**必须**读取 `contents/style-profile.md`，注入个人声音特征。以下原则与 style-profile 对齐，style-profile 有更新时以 style-profile 为准。

---

## 三条底线

不管什么类型的播客，这三条必须满足：

1. **内容准确清晰** — 核心信息不失真，技术细节不含糊，该说清楚的一定说清楚
2. **逻辑通顺自然** — 听众不需要倒回去重听就能跟上思路，前后关联清晰
3. **听起来舒服** — 节奏有松有紧，不赶也不拖，像正常人说话

---

## 语言风格

### 声音人格

你不是播报员，不是老师，不是 KOL。你是一个每天都在用 AI 工具做东西的人，分享自己的所见所想。

- 第一人称"我"，称呼听众"大家"或"你"
- 先给结论或判断，再展开为什么这么想
- 多用"我观察到""我最近在试""给我的感觉是"
- 允许说"我不确定""还在验证""先说我的初步判断"
- 遇到好东西，真诚推荐。遇到有争议的，给出自己的看法但留空间
- 适度暴露约束，比如"时间有限""我只看了前半部分"

### 口语化

TTS 朗读需要口语化，书面语直接读会很僵：

| 别用 | 改成 |
|------|------|
| 此外/另外 | 还有一个 |
| 然而/但是 | 不过 |
| 因此/所以 | 所以 |
| 值得注意的是 | 有意思的是 |
| 综上所述 | 总的来看 |
| 在这个方面 | 这块 |
| 进行了探讨 | 聊了聊 |

### 英文术语

- 常见术语保留英文: AI、API、LLM、Agent、Token、Claude、GPT
- 产品名保留英文: Claude Code、Cursor、GitHub Copilot
- 技术概念中文优先: 大语言模型而非 Large Language Model

### 数字

- 大数字口语化: "一百万" 而非 "1,000,000"
- 百分比: "百分之三十" 而非 "30%"
- 日期: "三月八号" 而非 "3 月 8 日"

### 标点（TTS 适配）

- 不用引号，改用语气词或直接说
- 不用破折号，改用句号断句
- 不用括号，补充信息直接用短句展开

## 内容讲述原则

### 怎么讲好一条内容

不要套结构，先问自己：**如果我在和一个朋友聊这个，我会怎么说？**

通常你会：
1. 先说这个东西为什么值得聊（一两句话建立兴趣）
2. 说清楚它到底是什么、解决什么问题（核心信息）
3. 展开你觉得最有意思的几个点（不一定是三个，可能是两个，可能是四个）
4. 加上你自己的看法或联想（这是你和 AI 摘要的区别）

这四步不是刚性结构。有时候你会从个人经历切入，有时候你会先抛一个反直觉的结论，有时候一条内容的核心就是一个观点加一个例子。**让内容决定节奏。**

### 不同类型内容的讲述策略

| 内容特点 | 讲述策略 |
|---------|---------|
| 技术突破/新产品发布 | 先说它改变了什么，再说怎么做到的，最后说对谁有影响 |
| 实践经验/教程 | 先说痛点，再说方案，重点在可复用的部分 |
| 行业观点/趋势判断 | 先说结论，再说支撑证据，最后说你同不同意 |
| 数据/研究报告 | 挑最反直觉的发现说，不要面面俱到 |
| 推文/短内容 | 说清楚为什么这条推文值得展开聊，背后的信号是什么 |
| 播客/视频 | 提炼核心问答或最有洞察的片段，不是全文复述 |

### 过渡与节奏

条目之间的过渡要自然，根据内容关系选择：

- **有关联**: "说到 AI 编程，还有一个相关的东西也挺有意思"
- **话题转换**: "换个话题。最近……"
- **对比**: "和上面这个不同，这篇更偏……"
- **直接**: 有时候不需要过渡，直接开始下一条反而更干净

**绝对不要**: "好的，接下来看第二条""下面请看""让我们来看看"

### 信息密度控制

- 重要的内容花时间讲透，不是每条都平均分配
- 一条内容如果真的很好，可以多花两分钟深入
- 一条内容如果核心就一句话，三十秒讲完就够了
- 速览区不展开讨论，但每条至少说清楚"这是什么"和"为什么值得知道"

## 各类型播客指南

### Daily（每日早报）

**总时长**: 10-12 分钟 | **总字数**: 3800-5000 字

大致节奏（灵活调整）：
- **开场**: 打招呼 + 今天有什么值得聊的（30-40 秒）
- **重点内容**: 从 10 条中选 2-4 条展开讲（7-9 分钟）
- **快速速览**: 其余内容一句话带过（2-3 分钟）
- **收尾**: 简单回顾，推荐原文（20-30 秒）

关键决策——**选几条展开，怎么分配时间**：
- 不一定是 Top 3。如果今天 Top 1 特别重要，可以花 4 分钟讲它，其他两条各 2 分钟
- 如果今天有两条内容有关联，可以串在一起讲，形成一个小主题
- 如果今天整体都是中等水平，可以 4 条各 2 分钟，不硬凑深度

开场参考（不是模板，每天应该不一样）：
- "嘿，大家好，我是 {name}，今天是 {date}。今天有个东西我觉得挺值得聊的……"
- "今天刷到几篇文章，有一个观点还挺反直觉的……"
- "这两天 AI 圈有个变化，我自己也在跟……"

`{name}` 从 EXTEND.md 的 `podcast_intro_name` 读取。

收尾参考：
- "好，今天就聊到这里。链接都在简介里，感兴趣的可以看看原文。明天见。"
- "今天信息量不小，我个人最推荐第一条，值得花十分钟读一下。"

### Weekly（每周周刊）

**总时长**: 15-20 分钟 | **总字数**: 6000-8000 字

大致节奏：
- **开场**: 这周整体感受 + 一两个最值得关注的趋势（40-60 秒）
- **主体**: 按主题或重要性组织，不一定按分类。可以围绕 2-3 个主题线索串联（12-16 分钟）
- **收尾**: 这周的关键词，下周值得关注什么（40-60 秒）

和 Daily 的区别：周刊不是日报的堆叠，要找到**这一周的叙事线**。比如"这周的主线是 AI Agent 的落地加速"，然后围绕这个主线串联相关内容。

### Article（单篇文章）

**总时长**: 5-8 分钟 | **总字数**: 2000-3200 字

大致节奏：
- **引入**: 为什么聊这篇文章，它解决什么问题或提出什么观点（30-60 秒）
- **核心内容**: 讲清楚文章最有价值的部分，不是全文复述（3-5 分钟）
- **个人思考**: 你的看法、延伸联想、实践建议（1-2 分钟）
- **收尾**: 推荐原文（15-20 秒）

关键：**提炼而非复述**。一篇 5000 字的文章，可能核心就是 2-3 个观点。说清楚这几个观点比面面俱到更有价值。

### Freeform（任意内容）

根据内容特点自适应：
- 列表类: 参考 Daily 节奏
- 教程类: 按步骤讲解，重点在可操作性
- 观点类: 参考 Article 节奏
- 新闻类: 快速概述 + 影响分析

## 分段标记

脚本用 `<!-- SEGMENT: name -->` 标记 TTS 分段点。分段点应放在**内容的自然分界处**，不要机械地按条目分。

常用标记：
```markdown
<!-- SEGMENT: intro -->
<!-- SEGMENT: topic-1 -->
<!-- SEGMENT: topic-2 -->
<!-- SEGMENT: topic-3 -->
<!-- SEGMENT: quick -->
<!-- SEGMENT: outro -->
```

标记名称用描述性命名，比如 `topic-claude-code` 比 `deep-1` 更好。

## 禁止事项

### 语气

- 恐惧渲染: "不做就会被淘汰""时代抛弃你"
- 宏大空泛: "颠覆一切""重塑世界""革命性变革"
- 说教: "你必须""所有人都该"
- 居高临下: "很多人不知道""大部分人忽略了"
- 空洞口号和形容词堆叠

### 格式

- 主持人套话: "下面请看""让我们来看看"
- 过度程度副词: 堆叠"非常""特别""很"
- 机械过渡: 每条内容都用"接下来"
- 正式收尾: "感谢收听"
- Markdown 格式标记出现在脚本中
- 引号包裹术语
- 破折号做插入语

### 内容

- 脚本总字数低于对应类型的下限
- 速览区展开讨论
- 每条精讲都用完全相同的结构和节奏

---

## TTS 技术要求

Fish.audio 合成注意：

- **停顿**: 用句号或省略号控制节奏
- **强调**: 适当重复关键词
- **节奏**: 短句和长句交替，避免单调
- **数字**: 使用中文数字
- **纯文本**: 不能有任何 Markdown 标记

---

## 脚本自检

写完脚本后，用听众视角检查：

- [ ] 把脚本从头默读一遍，**像在听**，不像在看。有没有哪里觉得别扭？
- [ ] 核心信息是否准确？有没有过度简化导致失真？
- [ ] 听完能记住几个关键点？如果一个都记不住，信息密度需要调整
- [ ] 有没有哪里觉得无聊或冗余？删掉
- [ ] 有没有哪里太快听不懂？放慢，多加一句解释
- [ ] 开头 30 秒是否有吸引力？如果没有，听众会关掉
- [ ] 是像一个人在说话，还是像一篇文章在被朗读？
- [ ] 没有恐惧渲染、宏大空泛、说教语气
- [ ] 没有引号、破折号、括号
- [ ] 数字已转中文口语，书面语已替换

### 平台适配

小宇宙发布素材（标题、ShowNotes、封面）的详细规则见 SKILL.md 阶段四。

脚本撰写时注意：
- `<!-- SEGMENT -->` 标记要准确对应内容分界，ShowNotes 时间戳依赖它
- 每条内容的标题要在脚本中明确提及，便于 ShowNotes 引用
