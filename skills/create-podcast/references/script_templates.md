# 播客脚本撰写模板与风格指南

## 整体定位

独白式内容播客，由主播（用户克隆声音）将各类内容转化为音频形式。

**核心气质**: 像一位懂技术的朋友在咖啡馆和你聊看到了什么有意思的东西——专业但不学术，有观点但不说教，轻松但有信息密度。

---

## 通用语言风格

### 总体基调

| 维度 | 规范 |
|------|------|
| **正式度** | 半正式，专业但不学术 |
| **人称** | 第一人称 "我"，称呼听众 "大家" 或 "你" |
| **语气** | 好奇、热情、有判断力。不是播报员，而是分享者 |
| **信息密度** | 高，每句话都有信息量，但节奏松弛有度 |

### 口语化处理

TTS 朗读需要口语化:

| 书面语 | 口语化 |
|--------|--------|
| 此外 | 另外 |
| 然而 | 不过 |
| 因此 | 所以 |
| 值得注意的是 | 有意思的是 |
| 综上所述 | 总的来看 |

### 英文术语处理

- **常见术语保留英文**: AI、API、LLM、Agent、Token、Claude、GPT
- **产品名保留英文**: Claude Code、Cursor、GitHub Copilot
- **技术概念中文优先**: 大语言模型（而非 Large Language Model）

### 数字与单位

- 大数字口语化: "一百万" 而非 "1,000,000"
- 百分比: "百分之三十" 而非 "30%"
- 日期: "三月八号" 而非 "3月8日"

## BCPT 结构（精讲核心模型）

所有深度内容遵循 **BCPT 结构**:

| 段落 | 内容 | 字数 | 说明 |
|------|------|------|------|
| **B**ackground | 介绍来源、作者、背景 | 100-150字 | 建立上下文 |
| **C**ore Problem | 要解决/讨论什么 | 60-100字 | 明确读者为什么要关注 |
| **P**oints | **3 个**核心观点或发现 | 350-550字 | 主体内容，每个观点配实例 |
| **T**akeaway | 个人评论或启发 | 100-150字 | 增加个人视角 |

⚠️ 观点部分必须覆盖 3 个观点。不足时补充行业影响分析、竞品对比或趋势预判。

**按内容类型调整**:

| 类型 | 背景介绍 | 核心素材 | 特殊处理 |
|------|---------|---------|---------|
| **ARTICLE** | 文章来源、作者 | `mainPoints` + 全文 | 标准 BCPT |
| **PODCAST** | 播客节目、嘉宾 | `autoChapters` + `questionsAnswers` | 可引用问答对 |
| **VIDEO** | 视频创作者、频道 | `mainPoints` + 描述 | 提及视频演示 |
| **TWITTER** | 推文作者(@用户名) | 推文文本 + 互动数据 | 互动数据佐证热度 |

## 模板一：Daily（每日早报）

**时长**: 10-12 分钟 | **字数**: 3800-5000 字（**不低于 3800 字**）

### 结构

```
开场白        ～40秒     品牌问候 + 日期 + 关键词 + 悬念预告
─────────────────────────
精讲 Top 1    ～3分钟     BCPT 结构
精讲 Top 2    ～2.5-3分钟 BCPT 结构
精讲 Top 3    ～2.5-3分钟 BCPT 结构
─────────────────────────
速览 4-10     ～3-4分钟   每条 20-30 秒
─────────────────────────
结尾          ～30秒     精华回顾 + 品牌引导
```

### 开场白模板（～40秒，120-180字）

```
大家好，我是 {name}，欢迎收听 BestBlogs 每日早报。今天是 {date}。

今天我们聊聊 {关键词1}、{关键词2} 和 {关键词3}。
{一句话预告最吸引人的内容}，我们马上开始。
```

`{name}` 从 EXTEND.md 的 `podcast_intro_name` 读取。预告句要具体有悬念。

### 精讲区（每条 2.5-3 分钟，650-1000 字）

遵循 BCPT 结构，精讲间用自然过渡:
- **关联式**: "说完了 AI 编程，我们看一个相关的话题——"
- **对比式**: "和上一条不同，这条更偏商业角度——"
- **禁止**: "好的，接下来看第二条"

### 速览区（每条 20-30 秒，至少 2 句话）

```
第{N}条，{标题}，来自{来源}。{核心要点}。{为什么值得关注}。
```

### 结尾模板（～30秒，80-120字）

```
以上就是今天的 BestBlogs 早报。
快速回顾一下，今天精讲了 {Top1}、{Top2} 和 {Top3}，
速览了 {关键词1}、{关键词2} 等七条内容。
希望这些内容对你有启发，我们明天见。
BestBlogs，遇见更好的技术阅读。
```

### 分段标记

```markdown
<!-- SEGMENT: intro -->
<!-- SEGMENT: deep-1 -->
<!-- SEGMENT: deep-2 -->
<!-- SEGMENT: deep-3 -->
<!-- SEGMENT: quick -->
<!-- SEGMENT: quick-2 -->
<!-- SEGMENT: outro -->
```

## 模板二：Weekly（每周周刊）

**时长**: 15-20 分钟 | **字数**: 6000-8000 字

### 结构

```
开场白        ～60秒     问候 + 本周概览 + 核心趋势
─────────────────────────
本周亮点      ～3分钟     3-5 个重要趋势/事件一句话概述
─────────────────────────
分类精选      ～8-10分钟  按类别（模型/开发/产品/资讯）各选 2-3 篇
  模型         ～2-3分钟   最重要的 1 篇 BCPT + 其余简述
  开发         ～2-3分钟   同上
  产品         ～2分钟     同上
  资讯         ～1-2分钟   快速过
─────────────────────────
重点深度      ～3-4分钟   从全部精选中选 1 篇做超深度 BCPT
─────────────────────────
总结          ～60秒     本周关键词 + 趋势预判 + 品牌结尾
```

### 开场白

```
大家好，我是 {name}，这里是 BestBlogs 每周精选。
本期我们覆盖 {date_range}，
这周最值得关注的是 {核心趋势}。
```

### 分段标记

```markdown
<!-- SEGMENT: intro -->
<!-- SEGMENT: highlights -->
<!-- SEGMENT: category-model -->
<!-- SEGMENT: category-dev -->
<!-- SEGMENT: category-product -->
<!-- SEGMENT: category-news -->
<!-- SEGMENT: deep-focus -->
<!-- SEGMENT: outro -->
```

## 模板三：Article（单篇文章）

**时长**: 5-8 分钟 | **字数**: 2000-3200 字

### 结构

```
开场白        ～30秒     简短引入文章和作者
─────────────────────────
深度解读      ～4-6分钟   扩展版 BCPT（更深入）
  背景         ～1分钟     作者背景 + 文章上下文 + 写作动机
  问题         ～30秒     核心问题
  观点         ～2-3分钟   3-5 个关键观点，每个配详细分析
  延伸         ～1分钟     与其他领域/文章的关联思考
─────────────────────────
个人思考      ～1分钟     更深入的个人启发和实践建议
─────────────────────────
结尾          ～20秒     推荐原文 + 品牌结尾
```

### 分段标记

```markdown
<!-- SEGMENT: intro -->
<!-- SEGMENT: deep -->
<!-- SEGMENT: reflection -->
<!-- SEGMENT: outro -->
```

## 模板四：Freeform（任意内容）

**时长**: 视内容而定 | **字数**: 视内容而定

根据内容的结构和特点，自适应组织脚本:

- **列表类内容**: 类似 Daily 模板，逐条介绍
- **教程类内容**: 按步骤讲解，重点在实操
- **观点类内容**: 类似 Article 模板，围绕核心论点展开
- **新闻类内容**: 快速概述 + 关键影响分析

### 分段标记

根据内容自然分段，使用描述性标记:

```markdown
<!-- SEGMENT: intro -->
<!-- SEGMENT: part-1 -->
<!-- SEGMENT: part-2 -->
<!-- SEGMENT: outro -->
```

## 禁止事项

- **禁止** 使用 "下面请看"、"让我们来看看" 等主持人套话
- **禁止** 过度使用 "非常"、"特别"、"很" 等程度副词
- **禁止** 每条内容都用 "接下来" 过渡
- **禁止** 结尾说 "感谢收听"（太正式）
- **禁止** 在脚本中使用 Markdown 格式标记（#、**、- 等）
- **禁止** 速览区展开讨论
- **禁止** 背景介绍超过 3 句话
- **禁止** Daily 模板总字数低于 3800 字
- **禁止** 脚本总字数超过对应模板上限

---

## TTS 优化标记

Fish.audio 支持的文本处理:

- **停顿**: 用 "。" 或 "……" 控制停顿节奏
- **强调**: 适当重复关键词，TTS 会自然加重
- **节奏**: 短句和长句交替，避免单调
- **数字**: 使用中文数字（"三个" 而非 "3个"）
