# 视频配音脚本风格指南

## 核心理念

视频配音不是播客的缩略版。画面承担了 40-50% 的信息传递，配音的职责是**引导注意力、补充语境、输出判断**。

**一句话定位**: 一个每天在做 AI 产品的人，边看画面边和你聊他的理解。

**风格锚点**: 脚本**必须**读取 `contents/style-profile.md`，注入个人声音特征。以下原则与 style-profile 对齐，style-profile 有更新时以 style-profile 为准。

---

## 视频 vs 播客：配音差异

| 维度 | 播客配音 | 视频配音 |
|------|---------|---------|
| 信息承载 | 100% 靠声音 | 50-60% 声音 + 40-50% 画面 |
| 描述性语言 | 需要口述图表、数据 | 画面展示，配音点到即可 |
| 字数密度 | ~400 字/分钟 | ~300 字/分钟（留给视觉消化） |
| 停顿 | 短停顿调节节奏 | 可以有 1-2 秒静默让画面说话 |
| 总字数 | 精讲 ~500 字/条 | 精讲 ~350 字/条 |

**核心差异**: 播客需要"说画面"，视频需要"说画面没有的"。

---

## 三条底线

与播客一致，不可妥协：

1. **内容准确清晰** — 核心信息不失真，技术细节不含糊
2. **逻辑通顺自然** — 观众不需要倒回去重看就能跟上
3. **观感舒适** — 节奏有松有紧，声画同步，没有等待感

---

## 语言风格

### 声音人格

与播客一致：你是一个每天都在用 AI 工具做东西的人，分享自己的所见所想。

- 第一人称"我"，称呼观众"大家"或"你"
- 先给结论或判断，再展开
- 允许不确定："还在验证""先说初步判断"
- 视频特有：可以用"你看这个""画面上这个"等指示性语言

### 口语化要求

同播客，书面语替换为口语。额外注意：

- 画面有文字时不要逐字念，用自己的话概括
- 看到数据图表说"你可以看到"而非复述每个数字
- 遇到代码截图说"这段代码的关键在于"而非念代码

### 标点（TTS 适配）

- 不用引号，改用语气词或直接说
- 不用破折号，改用句号断句
- 不用括号，补充信息直接用短句展开
- 英文术语保留英文：AI、API、LLM、Agent
- 数字用中文口语：一百万、百分之三十

## 视频模板配音指南

### Daily 每日视频

**目标时长**: 8-12 分钟 | **目标字数**: 2800-4200 字

配音节奏：

```
开场 (25-35秒, ~150字)
  → 打招呼 + 今天最值得聊的一句话钩子

精讲 2-3 条 (每条 2-4分钟, ~250-350字/条)
  → 每条：为什么值得看 → 核心发现 → 我的看法
  → 条目间自然过渡，不机械

速览 5-7 条 (2-3分钟, ~30-50字/条)
  → 每条一句话：是什么 + 为什么值得知道

收尾 (15-25秒, ~80字)
  → 今天最值得记住的一个点 + 推荐原文
```

**精讲选择策略**:
- 不一定选 Top 3。一条特别好可以花 4 分钟，其他各 2 分钟
- 有关联的内容可以串讲，形成小主题
- 整体中等时，多选几条各 2 分钟，不硬凑深度

**配音与画面配合**:
- cover slide 时说"今天第一条聊……"引入主题
- point slide 时说核心观点，画面展示配图和关键文字
- quote slide 时可以短暂停顿让观众读完金句，再加一句评论
- takeaway slide 时给出你的判断和思考

### Weekly 每周视频

**目标时长**: 12-18 分钟 | **目标字数**: 4200-6500 字

配音节奏：

```
开场 (35-50秒, ~200字)
  → 这周整体感受 + 一个贯穿主线

主题段落 2-4 个 (每段 3-5分钟)
  → 每段围绕一个主题串联相关内容
  → 段落间有明确的话题切换

收尾 (25-40秒, ~120字)
  → 这周关键词 + 下周值得关注什么
```

**与 Daily 的区别**: 周刊不是日报堆叠。找到这一周的叙事线，比如"这周主线是 Agent 落地加速"，围绕主线串联内容。

### Article 单篇视频

**目标时长**: 5-8 分钟 | **目标字数**: 1800-2800 字

配音节奏：

```
引入 (25-40秒, ~120字)
  → 为什么聊这个 + 它解决什么问题

核心内容 (3-5分钟, ~1200-1800字)
  → 讲清楚最有价值的 2-3 个点
  → 不是全文复述，是提炼

个人思考 (1-2分钟, ~400-600字)
  → 你的看法、延伸联想、实践建议

收尾 (10-15秒, ~50字)
  → 推荐原文
```

**关键**: 提炼而非复述。5000 字文章核心可能就 2-3 个观点，说清楚这几个比面面俱到更有价值。

### Freeform 自由内容

根据内容特点自适应：
- 列表类：参考 Daily 节奏
- 教程类：按步骤讲，重点在可操作性，配合截图画面
- 观点类：参考 Article 节奏
- 新闻类：快速概述 + 影响分析

## 节奏与停顿

### 信息密度曲线

视频要有节奏感。不能一直高密度（累），也不能一直低密度（无聊）。

```
密度
 高  ╱╲    ╱╲        ╱╲
    ╱  ╲  ╱  ╲      ╱  ╲
 低╱    ╲╱    ╲____╱    ╲___
   开场  精讲1  精讲2  速览  收尾
```

- **开场**: 中等密度，建立兴趣
- **精讲**: 高密度（核心观点）→ 低密度（消化时间）交替
- **精讲间过渡**: 短暂低密度呼吸
- **速览**: 均匀中低密度，快节奏
- **收尾**: 低密度，留下回味

### 声画同步停顿

配音脚本中用省略号 `……` 标记需要短停顿的位置：

- slide 切换时停顿 0.5 秒：让观众适应新画面
- 金句显示时停顿 1-2 秒：让观众读完再评论
- 话题切换时停顿 1 秒：心理分段
- 重要结论前停顿 0.5 秒：制造期待

### 过渡策略

条目之间的过渡要自然：

- **有关联**: "说到 AI 编程，还有一个相关的东西"
- **话题转换**: "换个话题。最近……"
- **对比**: "和这个不同，下一篇更偏……"
- **直接**: 有时不需要过渡，直接开始反而更干净

**绝对不要**: "好的，接下来看第二条""下面请看""让我们来看看"

## 分段标记

与播客一致，用 `<!-- SEGMENT: name -->` 标记 TTS 分段点。

视频特有的标记策略：
- 每个精讲条目独立分段（方便音视频对齐）
- 速览整体作为一个分段（节奏连贯）
- 开场和收尾各一个分段

```markdown
<!-- SEGMENT: intro -->
<!-- SEGMENT: deep-topic-name-1 -->
<!-- SEGMENT: deep-topic-name-2 -->
<!-- SEGMENT: quick-browse -->
<!-- SEGMENT: outro -->
```

标记名称用描述性命名，比如 `deep-mobile-agent` 比 `deep-2` 更好。

## 分享心态

视频的核心目的是**让观众从中获得信息和思考**，不是展示你知道多少。

### 信息传递检查

每条精讲必须回答：
1. **这是什么** — 一句话说清楚核心
2. **为什么重要** — 对谁有什么影响
3. **我的判断** — 你怎么看，有什么可行动的

速览必须回答：
1. **这是什么** — 一句话概括
2. **为什么值得知道** — 为什么进了今天的推荐

### 价值密度原则

- 每 30 秒至少输出一个有价值的信息点
- 避免重复说同一个观点的不同表述
- 如果某段没有新信息，砍掉
- 宁可少说一条，也不要每条都蜻蜓点水

---

## 禁止事项

### 语气

- 恐惧渲染："不做就会被淘汰""时代抛弃你"
- 宏大空泛："颠覆一切""重塑世界""革命性变革"
- 说教："你必须""所有人都该"
- 居高临下："很多人不知道""大部分人忽略了"

### 格式

- 主持人套话："下面请看""让我们来看看"
- 过度副词堆叠
- 机械过渡：每条都用"接下来"
- Markdown 格式标记出现在脚本中
- 引号、破折号、括号

### 内容

- 脚本总字数低于对应类型下限
- 逐字复述画面上的文字
- 速览区展开讨论
- 每条精讲都用完全相同的结构

---

## 脚本自检

写完脚本后检查：

- [ ] 默读一遍，像在看视频时听，有没有哪里觉得画面和声音脱节？
- [ ] 核心信息是否准确？过度简化导致失真？
- [ ] 看完能记住几个关键点？一个都记不住就要调整
- [ ] 有没有哪段配音和画面在说同一件事（冗余）？
- [ ] 有没有哪段画面在等配音说完（拖沓）？
- [ ] 开头 10 秒是否有吸引力？视频观众比播客更没耐心
- [ ] 像一个人在聊天，还是在念稿？
- [ ] 每条精讲都传递了"这是什么、为什么重要、我怎么看"？
- [ ] 没有恐惧渲染、宏大空泛、说教语气？
- [ ] 没有引号、破折号、括号？数字已转中文口语？
