# 视频视觉设计指南

## 整体定位

内容视频是将文字/音频内容转化为**视觉增强版**。画面与配音内容同步切换，每 8-15 秒切换一张 slide。视觉的核心目的是**帮助观众更好地理解和记住内容**。

**核心原则**:
- **声画同步**: slides 切换与配音一一对应，观众看到的就是听到的
- **节奏鲜明**: 高信息密度与视觉呼吸交替，不停滞不赶场
- **素材优先**: 优先使用原文配图，不足时 AI 生成
- **简洁克制**: 画面服务于内容，不喧宾夺主
- **无等待感**: 每张 slide 都有信息增量，没有空白等待
- **视觉多样**: 同类型视频也要有不同的视觉节奏，避免审美疲劳

## 品牌视觉体系

### 配色

| 角色 | 色值 | RGB | 用途 |
|------|------|-----|------|
| 墨蓝 | `#1a365d` | 26, 54, 93 | 主色、标题、品牌 |
| 琥珀 | `#d97706` | 217, 119, 6 | Top 1 唯一强调色 |
| 米白 | `#fefdfb` | 254, 253, 251 | 背景 |
| 深炭灰 | `#374151` | 55, 65, 81 | 正文文字 |
| 中灰 | `#6b7280` | 107, 114, 128 | 辅助信息 |
| 淡灰分隔 | `#e5e7eb` | 229, 231, 235 | 分隔线 |

### 字体

| 用途 | 字体栈 |
|------|--------|
| 标题/品牌 | Georgia, 'Noto Serif SC', serif |
| 正文/摘要 | -apple-system, 'PingFang SC', 'Noto Sans SC', sans-serif |
| 数字/评分 | 'Inter', 'SF Mono', monospace |

### 视觉气质

高端设计杂志（Monocle、Kinfolk）+ 瑞士国际主义排版:
- 大量留白，信息有呼吸感
- 无渐变、无发光、无粒子效果
- 平面设计，不堆叠阴影
- 克制的动效，以缓入缓出为主

## 视频结构（场景分解）

### 画面尺寸

- **横版**: 1920×1080 (16:9)，默认
- **竖版**: 1080×1920 (9:16)，可选参数

### 通用场景骨架

所有视频都有开场和结尾，中间部分由叙事策略决定：

```
[Scene 1] 品牌/标题开场     3-5秒
[Scene 2] 关键词展示        3-4秒（可选，适合 daily/weekly）
[Scene 3-N] 主体内容        ← 由叙事策略决定
[Scene Last] 品牌结尾       3-5秒
```

---

## 各场景设计

### 品牌/标题开场（3-5 秒）

显示视频标题和日期。标题不必是固定格式，根据内容生成有吸引力的标题。

```
┌──────────────────────────────────┐
│                                  │
│        {title}                   │  ← 墨蓝衬线体，居中
│        {date}                    │  ← 中灰，小号
│                                  │
└──────────────────────────────────┘
```

### 关键词（3-4 秒，可选）

关键词标签依次淡入（每个间隔 6 帧）。4-6 个关键词。

**使用建议**: daily/weekly 用关键词让观众快速了解内容；article/实操可以跳过这个场景直接进入主体。

### 品牌结尾（3-5 秒）

显示品牌名和标语。仅淡入，无淡出。

## 6 种 Slide 类型

| 类型 | 布局 | 用途 | 信息密度 |
|------|------|------|---------|
| `cover` | 全幅大图 + 底部暗色渐变 + 标题 | 引入 | 低（视觉吸引） |
| `source-card` | 白色卡片 UI + 评分徽章 | 来源预览 | 中 |
| `problem` | 居中大字 | 核心问题/悬念 | 低（制造悬念） |
| `point` | 左图(60%) + 右文(40%) | 观点/信息 | 高（核心信息） |
| `quote` | 斜体居中 + 装饰引号 | 金句/呼吸 | 低（消化空间） |
| `takeaway` | 暗色叠加 + 白字 | 总结/判断 | 中（收束思考） |

### Slide 文字排版规范

slides 上的文字同样要遵循排版和表达规范：

**标点清理**: slide 文字中不出现引号、破折号、括号。原文有这些标点时，改写为更直接的表达。

**CJK 排版**: 中文与英文之间加空格（`Claude Code 的新功能`），中文与数字之间加空格（`超过 500 万用户`）。

**简洁表达**: slide 文字比配音更精炼。一张 slide 最多一个核心观点，用短句表达，避免长难句。`problem` slide 的问题要一眼能读懂，`point` slide 的观点要一句话说清。

**信息准确**: 简化表达不丢失核心信息。数字和结论必须与源文章一致。

### Top 1 特殊处理

Cover slide 序号使用琥珀色，整体视觉稍强调。

### Ken Burns 效果

含图片的 slide 应用缓慢放大（1.0 → 1.05），增加画面动感。

### 无图片 Fallback

- `cover`/`point`/`takeaway` 无图时退化为纯色背景 + 居中大字
- 无 `slides` 数据时自动从 `points`/`quote`/`summary` 生成等分 slides

## Slides 编排：开场变体

不是每条精讲都要以 `cover` 开始。根据内容和叙事策略灵活选择开场 slide。

### 变体 A：封面先行（经典）

```
cover → source-card → problem → point...
```
适用：有高质量封面图的内容，标准引入。

### 变体 B：问题先行

```
problem → cover → point...
```
适用：先抛出一个引人思考的问题，再揭示来源。适合深度聚焦、对比分析。

### 变体 C：金句先行

```
quote → cover → point...
```
适用：内容有一句特别吸引人的话，先放出来吊胃口。

### 变体 D：观点先行

```
point → cover → point...
```
适用：核心观点本身就是最好的钩子，直接亮结论再展开。实操拆解常用。

### 变体 E：无封面直入

```
problem → point → point...
```
适用：内容无好图或速度感更重要时，跳过封面直接进入信息。

**选择原则**: 同一期视频中，不同精讲条目可以用不同的开场变体，增加视觉节奏变化。

## Slides 编排：节奏规则

### 信息密度波动

slide 类型的排列要形成**信息密度波动**：

```
信息密度
 高     ┃         ┃     ┃
 中  ┃  ┃    ┃    ┃  ┃  ┃  ┃
 低  ┃  ┃  ┃ ┃  ┃ ┃  ┃  ┃  ┃
    cover src prob pt  pt  qt  pt  tkwy tkwy
    引入 → 背景 → 悬念 → 观点 → 呼吸 → 深入 → 收束
```

### 排列约束

- 连续 `point` 不超过 3 张（之后必须插入低密度 slide）
- `quote` 后接 `point` 或 `takeaway`（呼吸后继续或收束）
- `takeaway` 放最后 1-2 张
- 每张 slide 8-15 秒，绝不超过 20 秒
- `durationRatio` 之和必须 = 1.0

### 各条目 Slides 数量参考

| 条目类型 | Slides 数量 | 说明 |
|---------|------------|------|
| 标准精讲 | 6-10 张 | 2-4 分钟 |
| 深度聚焦 | 12-15 张 | 4-6 分钟 |
| 主题串联段 | 8-12 张 | 3-5 分钟 |
| 对比分析方 | 5-8 张 | 2-3 分钟 |
| 速览卡片 | 1 张/条 | 15-20 秒/条 |

## 各叙事策略的视觉编排

### 策略一：标准精讲 + 速览

精讲条目各自独立，slides 编排标准。精讲间用不同的开场变体增加变化。

```
精讲 1: cover → source-card → problem → point×2-3 → quote → takeaway
精讲 2: problem → cover → point×2 → takeaway  ← 用不同开场
精讲 3: point → quote → takeaway               ← 精简版
速览:   quick-card × 5-7
```

### 策略二：深度聚焦

单条大量 slides，视觉叙事要有层次递进：

```
问题先行: problem → cover → source-card
层层展开: point → point → quote → point → point → quote
收束:     takeaway × 2
```

**关键**: 15 张 slides 不能视觉单调。每张 `point` 配不同图片，`quote` 作为视觉呼吸分隔"段落"。

### 策略三：主题串联

slides 要体现"串联感"——同一主题下不同来源的内容交织：

```
主题封面: cover（用主题关键词，非文章标题）
来源 A:   point × 2（A 的核心观点）
过渡:     problem 或 quote（从 A 过渡到 B 的桥梁）
来源 B:   point × 2（B 的补充/深入）
综合:     takeaway（这个主题的整体判断）
```

### 策略四：对比分析

视觉上要让观众感受到"两个视角"：

```
开场:   problem（抛出对比的核心问题）
A 方:   cover(A) → point × 2-3
过渡:   quote 或 problem（"但另一方的观点是……"）
B 方:   cover(B) → point × 2-3
综合:   takeaway × 1-2（你的判断和边界条件）
```

### 策略五：趋势信号

快节奏列举 + 深入展开的双段式视觉：

```
开场:     problem（趋势判断的一句话）
信号列举: point × 3-5（快节奏，每张 8-10 秒，列证据）
深入:     cover → point × 3-4 → quote（挑 1-2 个信号展开）
判断:     takeaway × 2
```

### 策略六：实操拆解

截图和示意图为主的视觉：

```
痛点:   problem（痛点描述）
背景:   point（之前的做法/为什么需要）
步骤:   point × 4-6（每步配截图/示意图）
效果:   point（效果展示）
总结:   takeaway（建议和局限性）
```

## 速览场景

速览引导文字 + 速览卡片逐张切换。

每张卡片包含：序号、标题、一句话摘要、来源。

**引导词变化**: 不是每次都用"快速速览"。可以用"还有几条值得看看""更多今天的亮点""快速过一下其他内容"。

**节奏要求**: 速览卡片节奏紧凑均匀，每张 15-20 秒，不展开讨论。

## 动效规范

### 基础动效

- **淡入为主**: opacity 0→1 + translateY 10px→0
- **缓动函数**: spring-based 或 ease-out
- **时长**: 0.4-0.8s，不超过 1s
- **Slide 间过渡**: 0.8s 交叉淡入淡出（fade），不用其他转场

### 节奏辅助动效

- **关键词**: 标签依次淡入（每个间隔 6 帧 = 0.2s）
- **速览卡片**: 均匀切换，无额外装饰动效
- **精讲 slides**: 交叉淡入淡出，过渡期间音频不中断

### 禁止的动效

- 翻页、旋转、弹性、缩放等花哨转场
- 文字逐字出现（打字机效果）
- 元素从屏幕外飞入
- 任何持续超过 1s 的过渡

## 图片处理

### 尺寸与裁剪

- 统一处理为 16:9 比例，使用 `object-fit: cover`
- 建议渲染分辨率: 图片区域 1152×648

### 质量要求

- 最小宽度: 800px
- 格式: JPG/PNG/WebP
- 文件大小: ≥10KB

### AI 生成图规范

生成图是源文章真实图片的**替补**，不是首选。使用时必须确保与视频整体视觉品质一致。

#### 风格基线

统一风格前缀（所有生成图 prompt 开头加上）：

```
Minimalist tech illustration, flat design, muted color palette
(ink blue #1a365d, cream #fefdfb, gray tones), clean lines,
no text overlays, 16:9 aspect ratio.
```

#### 按 Slide 类型定制 prompt

不同 slide 类型需要不同的图片构图：

| Slide 类型 | 图片需求 | prompt 策略 |
|-----------|---------|-----------|
| `cover` | 全幅大图，视觉冲击力 | 聚焦主题的概念性场景，留出底部空间给标题渐变 |
| `point` | 左侧 60% 配图 | 具体概念的图解或示意，主体偏左或居中，右侧可留白 |
| `takeaway` | 暗色叠加底图 | 偏暗调或有纹理的抽象图，叠加半透明后文字仍清晰 |

#### prompt 内容要求

**必须**: prompt 描述该 slide 实际讲述的具体概念

```
✅ 好的 prompt：
"...A split-view diagram showing a monolithic LLM agent on the left
transforming into three specialized micro-agents on the right,
connected by message arrows..."

❌ 差的 prompt：
"...An AI technology concept with neural network and data flow..."
```

**关键原则**:
- 每张生成图的主体必须不同（不能 3 张图都是"电脑屏幕上的代码"）
- 概念要具体到当前文章的论点，不能是泛泛的"AI/技术"
- 避免生成人物面部（容易不自然）
- 避免生成文字（AI 文字渲染质量差）

#### 与真实图片的融合

当同一条目中既有真实图片又有生成图时：

- 生成图的色调参考真实图片的整体调性（冷色/暖色/中性）
- 如果真实图片偏写实（如产品截图），生成图不要过于抽象
- 如果真实图片偏图解（如架构图），生成图也用图解风格
- 生成图的复杂度不应高于真实图片，避免喧宾夺主

#### 质量门控

生成后必须检查：
- [ ] 与当前 slide 讲述的内容直接相关（不是泛泛的概念图）
- [ ] 与同条目中其他图片的风格、色调不冲突
- [ ] 没有不自然的 AI 痕迹（畸变的文字、不合理的结构）
- [ ] 缩放到 1920×1080 后仍清晰，没有明显模糊
- [ ] 在 Ken Burns 效果（1.0→1.05 放大）下边缘不会露出空白

不合格的生成图不如不放。**纯色背景 + 大字排版的 fallback 永远好于一张无关或低质的图。**

---

## 观看体验核心检查

视频渲染前对照检查：

- [ ] 没有连续超过 20 秒停留在同一画面
- [ ] slides 切换时配音有自然过渡（不是句子说到一半突然换画面）
- [ ] 高密度 `point` slides 之间有 `quote` 或 `problem` 做呼吸
- [ ] 同一期视频中不同精讲条目使用了不同的开场变体
- [ ] 速览段节奏均匀，不忽快忽慢
- [ ] 开场 10 秒内有视觉吸引力（不是纯文字空白）
- [ ] 图片质量统一，没有明显低质图混入
- [ ] 品牌开场和结尾简洁，不占过多时长
- [ ] 和上一期视频的视觉节奏有差异

---

## 设计禁忌

- **不要** 使用蓝紫渐变科技感
- **不要** 添加发光、粒子、毛玻璃效果
- **不要** 给卡片加投影
- **不要** 使用超过两种强调色（墨蓝 + 琥珀）
- **不要** 画面塞满信息（每张 slide 最多一个核心观点）
- **不要** 文字动画过快或过花哨
- **不要** 速览区使用和精讲一样复杂的布局
- **不要** 连续 4+ 张高信息密度 slides
- **不要** 每条精讲都用完全相同的 slides 开场方式

## 平台适配

### B站

- 默认横版 16:9
- 封面图: 从视频关键帧或精讲封面图中选取，叠加标题文字
- 前 3 秒必须有视觉吸引力，B站用户快速滑动，开场不能太慢
- 时间戳导航写入描述

### YouTube

- 同 B站，横版 16:9
- 缩略图单独生成，大字 + 对比色 + 简洁画面
- 英文字幕可选

### 微信视频号

- 可选竖版 9:16，Remotion 渲染参数 `1080×1920`
- 竖版布局调整: 图片占上半部，文字占下半部
- 时长建议 3-5 分钟（视频号偏好短内容）
