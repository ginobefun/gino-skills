# 视频视觉设计指南

## 整体定位

内容视频是将文字/音频内容转化为**视觉增强版**。画面与配音内容同步切换，每 8-15 秒切换一张 slide。

**核心原则**:
- **画面丰富**: 避免长时间停留在同一画面
- **音视频同步**: slides 内容与配音一一对应
- **素材优先**: 优先使用原文配图，不足时 AI 生成
- **简洁克制**: 画面服务于内容，不喧宾夺主

## 品牌视觉体系

### 配色

| 角色 | 色值 | RGB | 用途 |
|------|------|-----|------|
| 墨蓝 | `#1a365d` | 26, 54, 93 | 主色、标题、品牌 |
| 琥珀 | `#d97706` | 217, 119, 6 | Top 1 唯一强调色 |
| 米白 | `#fefdfb` | 254, 253, 251 | 背景 |
| 深炭灰 | `#374151` | 55, 65, 81 | 正文文字 |
| 中灰 | `#6b7280` | 107, 114, 128 | 辅助信息 |
| 淡灰分隔 | `#e5e7eb` | 229, 231, 235 | 分隔线 |

### 字体

| 用途 | 字体栈 |
|------|--------|
| 标题/品牌 | Georgia, 'Noto Serif SC', serif |
| 正文/摘要 | -apple-system, 'PingFang SC', 'Noto Sans SC', sans-serif |
| 数字/评分 | 'Inter', 'SF Mono', monospace |

### 视觉气质

高端设计杂志（Monocle、Kinfolk）+ 瑞士国际主义排版:
- 大量留白，信息有呼吸感
- 无渐变、无发光、无粒子效果
- 平面设计，不堆叠阴影
- 克制的动效，以缓入缓出为主

## 视频结构（场景分解）

### 画面尺寸

- **横版**: 1920×1080 (16:9)，默认
- **竖版**: 1080×1920 (9:16)，可选参数

### 场景流程

```
[Scene 1] 品牌/标题开场     3-5秒
[Scene 2] 关键词展示        3-4秒
[Scene 3-N] 精讲项 Slides   对应音频时长
[Scene N+1] 速览引导        2-3秒
[Scene N+2] 速览卡片组      对应音频时长
[Scene Last] 品牌结尾       3-5秒
```

场景数量根据内容动态调整。单篇文章可能只有 1 个精讲无速览。

---

## 各场景设计

### Scene 1: 品牌/标题开场（3-5 秒）

显示视频标题（`title` 字段）和日期。可选显示品牌名。

```
┌──────────────────────────────────┐
│                                  │
│        {title}                   │  ← 墨蓝衬线体，居中
│        {date}                    │  ← 中灰，小号
│                                  │
└──────────────────────────────────┘
```

### Scene 2: 关键词（3-4 秒）

关键词标签依次淡入。

### Scene 3-N: 精讲场景 — Slides 模型

每条精讲由 **8-12 张 slides** 组成，slides 间用 0.8s 交叉淡入淡出。

#### 6 种 Slide 类型

| 类型 | 布局 | 用途 |
|------|------|------|
| `cover` | 全幅大图 + 底部暗色渐变 + 标题 | 引入 |
| `source-card` | 白色卡片 UI | 来源预览 |
| `problem` | 居中大字 | 核心问题 |
| `point` | 左图(60%) + 右文(40%) | 观点 |
| `quote` | 斜体居中 + 装饰引号 | 金句 |
| `takeaway` | 暗色叠加 + 白字 | 总结 |

#### 标准 Slides 编排示例（150 秒精讲）

```
[cover]        ~15s  全幅封面图 + 标题
[source-card]  ~10s  来源预览卡片
[problem]      ~12s  居中问题文字
[point]        ~20s  观点 1 + 配图
[point]        ~15s  观点 1 补充
[point]        ~20s  观点 2 + 配图
[point]        ~18s  观点 3 + 配图
[quote]        ~10s  金句引用
[takeaway]     ~15s  个人思考
[takeaway]     ~15s  总结要点
```

#### Top 1 特殊处理

Cover slide 序号使用琥珀色，整体视觉稍强调。

#### Ken Burns 效果

含图片的 slide 应用缓慢放大（1.0 → 1.05）。

#### 无图片 Fallback

- `cover`/`point`/`takeaway` 无图时退化为纯色背景 + 居中大字
- 无 `slides` 数据时自动从 `points`/`quote`/`summary` 生成等分 slides

### 速览场景

速览引导（"快速速览"文字）+ 速览卡片逐张切换。

每张卡片包含：序号、标题、一句话摘要、来源。

### 品牌结尾（3-5 秒）

显示品牌名和标语。仅淡入，无淡出。

---

## 动效规范

- **淡入为主**: opacity 0→1 + translateY 10px→0
- **缓动函数**: spring-based 或 ease-out
- **时长**: 0.4-0.8s，不超过 1s
- **无花哨转场**: 不使用翻页、旋转、弹性等效果

---

## 图片处理

### 尺寸与裁剪

- 统一处理为 16:9 比例，使用 `object-fit: cover`
- 建议渲染分辨率: 图片区域 1152×648

### 质量要求

- 最小宽度: 800px
- 格式: JPG/PNG/WebP
- 文件大小: ≥10KB

### AI 生成图风格

```
Minimalist tech illustration, flat design, muted color palette
(ink blue #1a365d, cream #fefdfb, gray tones), clean lines,
no text overlays, 16:9 aspect ratio.
```

同一期所有生成图风格一致。

---

## 设计禁忌

- **不要** 使用蓝紫渐变科技感
- **不要** 添加发光、粒子、毛玻璃效果
- **不要** 给卡片加投影
- **不要** 使用超过两种强调色
- **不要** 画面塞满信息
- **不要** 文字动画过快或过花哨
- **不要** 速览区使用和精讲一样复杂的布局

---

## 平台适配

### B站

- 默认横版 16:9
- 封面图: 从视频关键帧或精讲封面图中选取，叠加标题文字
- 前 3 秒必须有视觉吸引力，B站用户快速滑动，开场不能太慢
- 时间戳导航写入描述

### YouTube

- 同 B站，横版 16:9
- 缩略图单独生成，大字 + 对比色 + 简洁画面
- 英文字幕可选

### 微信视频号

- 可选竖版 9:16，Remotion 渲染参数 `1080×1920`
- 竖版布局调整: 图片占上半部，文字占下半部
- 时长建议 3-5 分钟（视频号偏好短内容）
