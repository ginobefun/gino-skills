# 每日内容管理 (Daily Content Management)

编排每日内容全流程：数据获取 → 20 个选题生成 → 10 个内容创作 → 多渠道适配分发。核心理念是「一鱼多吃」—— 一个内容适配多个渠道，而非为每个渠道独立创作。

个人写作风格和兴趣领域见用户画像（`gino-bot/USER.md`），按需加载。

## 每日产出目标

| 维度 | 数量 | 说明 |
|------|------|------|
| 选题候选 | 20 个 | 从 BestBlogs + XGo 数据中筛选 |
| 内容创作 | 10 个 | 用户从 20 个选题中选择 |
| 长文博客 | 1 篇 | 1500-3000 字，有特殊创作流程 |
| 中等内容 | 3 个 | 推文串/小红书/公众号短文 |
| 短内容 | 6 个 | 推文/即刻/朋友圈 |

## 工作流概览

```
- [ ] 阶段一: 数据获取 — 调用 daily-content-curator 获取素材池
- [ ] 阶段二: 选题生成 — AI 生成 20 个选题候选（含角度和渠道建议）
- [ ] 阶段三: 用户选题 ⚠️ REQUIRED — 用户从 20 个中选择 ~10 个
- [ ] 阶段四: 内容创作 — 逐个生成内容，每个内容适配多渠道
- [ ] 阶段五: 审阅确认 ⚠️ REQUIRED — 用户审阅每个内容的多渠道版本
- [ ] 阶段六: 分发执行 ⛔ BLOCKING — 确认后调用发布 skills
- [ ] 阶段七: 记录归档 — 保存当日内容计划和产出
```

---

## 阶段一: 数据获取

调用 `daily-content-curator` skill 获取今日素材池。如用户已经执行过 daily-content-curator，直接读取当日已有的 curation 文件：

```
contents/daily-curation/YYYY-MM-DD/curation.md
```

**若文件不存在**，提示用户先运行 daily-content-curator，或询问是否现在执行。

同时读取以下补充数据源（如存在）：
- `contents/bestblogs-digest/YYYY-MM-DD/digest.txt` — BestBlogs 日报
- `contents/twitter-digest/digest-YYYY-MM-DD.md` — Twitter 摘要日报

### 补充：读取历史内容计划

```bash
# 读取最近 3 天的内容计划，用于避免选题重复
# contents/daily-content-plan/YYYY-MM-DD/plan.md
```

## 阶段二: 选题生成

基于阶段一的素材池，AI 生成 **20 个选题候选**。

### 选题维度覆盖

确保 20 个选题覆盖以下领域（按用户关注优先级）：

| 领域 | 目标占比 | 选题数 |
|------|---------|--------|
| AI / AI Coding / LLM | 30-40% | 6-8 个 |
| 编程开发 / 开发者工具 | 20-25% | 4-5 个 |
| 产品 / 设计 | 10-15% | 2-3 个 |
| 商业科技 / 创业 | 10-15% | 2-3 个 |
| 个人成长 / 思考 | 10% | 2 个 |

### 选题类型分类

每个选题标注**内容类型**，决定创作方式和渠道适配：

| 类型 | 字数 | 适配渠道 | 每日目标 |
|------|------|---------|---------|
| 🔴 长文 | 1500-3000 字 | 博客 + 公众号（同步） | 1 篇 |
| 🟡 中文 | 500-1500 字 | 推文串 / 小红书 / 公众号短文 | 3 个 |
| 🟢 短文 | 50-280 字 | 推文 / 即刻 / 朋友圈 | 6 个 |

### 选题卡片格式

```markdown
## 📋 今日选题候选 | YYYY-MM-DD

> 从 N 篇文章 + M 条推文中提取 20 个创作选题

---

### 选题 01 🔴 长文 | AI
**标题**: [建议标题]
**角度**: [创作切入角度 — 个人经验/观点解读/教程/趋势分析]
**素材来源**: [原始文章/推文标题 + 链接]
**核心观点**: [1-2 句概括要写什么]
**渠道适配**: 博客 + 公众号 → 可拆为 2 条推文 + 1 条即刻
**创作难度**: ⭐⭐⭐

---

### 选题 02 🟡 中文 | 编程
**标题**: [建议标题]
**角度**: [切入角度]
**素材来源**: [来源]
**核心观点**: [要写什么]
**渠道适配**: 推文串(5条) + 小红书 → 可拆为 1 条推文摘要
**创作难度**: ⭐⭐

---

### 选题 03 🟢 短文 | AI
**标题**: [建议标题]
**角度**: [观点/转评/分享]
**素材来源**: [来源]
**核心观点**: [一句话]
**渠道适配**: 推文 + 即刻 + 朋友圈
**创作难度**: ⭐

---
...（共 20 个）

## 📊 选题统计

| 类型 | 数量 | 领域分布 |
|------|------|---------|
| 🔴 长文 | 3 个 | AI×2, 编程×1 |
| 🟡 中文 | 7 个 | AI×3, 产品×2, 商业×2 |
| 🟢 短文 | 10 个 | AI×4, 编程×2, 产品×2, 个人×2 |
```

### 选题生成规则

1. **个人视角优先**: 优先选择能结合个人经验和观点的素材
2. **避免搬运**: 不直接翻译/转述原文，必须有独特角度
3. **热点敏感**: 重大行业新闻/产品发布优先处理
4. **历史去重**: 与最近 3 天的选题计划对比，避免重复话题
5. **一鱼多吃标注**: 长文/中文选题标注可拆解的短内容数量

## 阶段三: 用户选题确认

⚠️ **REQUIRED** — 必须等待用户选择

展示 20 个选题后，引导用户选择：

```
请从以上 20 个选题中选择你今天要创作的内容：

建议配额：1 个长文 + 3 个中文 + 6 个短文 = 10 个内容

你可以：
- 按编号选择（如 "01, 03, 05, 08, 11, 13, 15, 17, 18, 20"）
- 调整类型（如 "05 改为短文"）
- 修改角度（如 "03 从个人经验角度写"）
- 补充选题（如 "加一个关于 XX 的短文"）
- 调整渠道（如 "08 不发小红书"）
```

用户确认后，锁定最终选题清单，进入创作阶段。

## 阶段四: 内容创作

### 创作顺序

按照内容类型从长到短依次创作：

1. **长文博客**（1 篇）— 耗时最长，优先处理
2. **中等内容**（3 个）— 逐个创作
3. **短内容**（6 个）— 可批量生成

### 4.1 长文博客创作流程

长文有特殊流程，需要深度素材理解：

1. **素材深读**: 如需深度分析，调用 `deep-reading` skill 对核心素材进行分析
2. **大纲生成**: 生成博客大纲，⚠️ 等用户确认
3. **正文创作**: 调用 `content-synthesizer` 生成博客全文
4. **多渠道适配**: 从博客内容衍生公众号版本（段落更短、加粗关键词）

详细的博客和公众号写作规范见 `references/content-types.md`。

### 4.2 中等内容创作

每个中等内容一次创作，自动适配多渠道：

```markdown
## 🟡 选题 05: [标题]

### 主版本 — 推文串 (5 条)
1/5 [Hook — 核心观点]
2/5 [论点 1]
3/5 [论点 2]
4/5 [论点 3 + 数据]
5/5 [总结 + 互动引导]

### 适配版本 — 小红书
[标题 + emoji]
[正文 300-800 字，列表化]
#标签1 #标签2 ...

### 适配版本 — 即刻
[200-500 字讨论体]
```

### 4.3 短内容创作

短内容可批量生成，每个内容自动适配多渠道：

```markdown
## 🟢 选题 15: [标题]

### 主版本 — 推文
[280 字符以内]

### 适配版本 — 即刻
[同内容，加互动提问]

### 适配版本 — 朋友圈
[同内容，口语化调整]
```

### 一鱼多吃规则

核心原则：**先创作主版本，再适配其他渠道**，而非为每个渠道独立创作。

| 主版本 | 可适配渠道 | 适配方式 |
|--------|-----------|---------|
| 博客长文 | 公众号、知乎 | 段落重排 + 风格调整 |
| 推文串 | 小红书、即刻 | 合并为列表体/讨论体 |
| 单条推文 | 即刻、朋友圈 | 口语化 + 加互动 |
| 小红书 | 即刻 | 去标签 + 加讨论感 |

渠道详细写作规范见 `references/content-types.md`。

## 阶段五: 审阅确认

⚠️ **REQUIRED** — 每个内容创作完成后展示给用户审阅

### 审阅展示格式

```markdown
---
## ✅ 内容 1/10 — 🔴 长文: [标题]

### 📝 博客版本
[完整内容]

### 📱 公众号适配版
[完整内容]

---

你可以：
- ✏️ 修改内容（如 "第二段改一下"）
- 🔄 换个角度重写
- ✅ 确认通过，进入下一个
- ⏭️ 跳过，稍后再改
---
```

逐个展示，用户确认一个后再展示下一个。用户也可以选择「批量审阅」一次看所有。

## 阶段六: 分发执行

⛔ **BLOCKING** — 所有发布操作必须用户明确确认

### 分发清单

用户审阅完所有内容后，生成分发清单：

```markdown
## 📤 今日分发清单

### 自动发布（需确认）
| # | 内容 | 渠道 | 发布 Skill |
|---|------|------|-----------|
| 1 | [长文标题] | 博客 | 保存 Markdown |
| 2 | [长文标题] | 公众号 | post-to-wechat |
| 3 | [推文串标题] | Twitter | post-to-x |
| 4 | [短文标题] | Twitter | post-to-x |
| ... | ... | ... | ... |

### 手动发布（输出内容供复制）
| # | 内容 | 渠道 |
|---|------|------|
| 5 | [小红书标题] | 小红书 |
| 6 | [即刻标题] | 即刻 |
| 7 | [朋友圈标题] | 朋友圈 |

确认要发布哪些？可以：
- "全部发布" — 自动发布项逐个执行 + 手动项输出内容
- 按编号选择（如 "先发 1, 3, 4"）
- "稍后发布" — 仅保存，不立即发布
```

### 发布执行规则

1. **写操作安全**: 每批最多 5 个发布操作，逐批确认
2. **推文发布**: 调用 `post-to-x` skill
3. **公众号发布**: 调用 `post-to-wechat` skill
4. **博客保存**: 保存到 `contents/blog-posts/YYYY-MM-DD-{slug}.md`
5. **手动渠道**: 输出格式化好的内容，用户自行复制发布
6. **单个失败不中断**: 记录 ❌ 和错误信息，继续下一个
7. **连续失败 3 次暂停**: 告知用户可能是系统性问题

## 与其他 Skill 的协作

### 上游（数据获取）

| Skill | 用途 | 调用时机 |
|-------|------|---------|
| daily-content-curator | 获取今日素材池 | 阶段一 |
| bestblogs-daily-digest | 补充 BestBlogs 日报数据 | 阶段一（已有则读取） |
| xgo-digest-tweets | 补充 Twitter 摘要数据 | 阶段一（已有则读取） |

### 中游（内容创作）

| Skill | 用途 | 调用时机 |
|-------|------|---------|
| deep-reading | 长文素材深度分析 | 阶段四（长文创作前） |
| content-synthesizer | 内容生成引擎 | 阶段四（所有内容创作） |
| image-gen / cover-image | 生成配图/封面 | 阶段四（按需） |
| article-illustrator | 长文配图 | 阶段四（长文创作中） |

### 下游（发布分发）

| Skill | 用途 | 调用时机 |
|-------|------|---------|
| post-to-x | 发布推文/推文串 | 阶段六 |
| post-to-wechat | 发布公众号文章 | 阶段六 |

### 反馈闭环

| Skill | 用途 | 调用时机 |
|-------|------|---------|
| content-analytics | 分析内容表现 | 每周/月执行 |
| → content-strategy.md | 策略调整 | 反馈到阶段二选题权重 |

## 快捷启动

用户可以用简短指令触发特定阶段：

| 指令 | 效果 |
|------|------|
| "今天的内容" / "开始" | 完整流程从阶段一开始 |
| "继续" | 从上次中断处继续 |
| "看看选题" | 仅执行阶段一二，生成选题 |
| "开始写 XX" | 直接进入阶段四，创作指定选题 |
| "发布" | 进入阶段六，发布已审阅内容 |
| "今天写了啥" | 显示当日 plan.md 状态 |

---

## 错误处理

- **daily-content-curator 未执行**: 提示用户先执行，或询问是否现在运行
- **素材池为空**: 建议扩大时间范围，或使用用户自定义选题
- **content-synthesizer 生成质量不够**: 保留草稿，让用户提供更多角度/观点后重写
- **发布 skill 失败**: 输出内容文本供手动复制，不中断其他发布
- **USER.md 不可访问**: 使用通用风格，告知用户"未加载个人画像"
- **中断恢复**: 每个阶段完成后自动保存进度到 plan.md，中断后可从最近完成的阶段恢复
