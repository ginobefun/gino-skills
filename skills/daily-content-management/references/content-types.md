# 内容类型与渠道适配参考

本文档定义每种内容类型的详细写作规范、渠道特性和适配规则。SKILL.md 中的创作流程引用此文档。

---

## 一、内容类型定义

### 🔴 长文 (1500-3000 字)

**每日目标**: 1 篇
**创作时间**: 30-60 分钟（含深度阅读）
**核心价值**: 深度思考、系统性观点、个人经验

**创作流程**:
1. 深度阅读素材（可调用 deep-reading）
2. 生成大纲 → 用户确认
3. 正文创作 → 用户审阅
4. 适配公众号版本

**质量标准**:
- 必须有个人视角和真实经验
- 论点清晰，有数据/案例支撑
- 信息密度适中，每句话有信息量
- 不使用套话开头（"在当今..."、"随着...的发展"）

### 🟡 中篇内容 (500-1500 字)

**每日目标**: 3 个
**创作时间**: 10-20 分钟/个
**核心价值**: 聚焦单一观点，有一定展开

**适合主题**:
- 工具/产品使用心得
- 技术概念解读
- 行业趋势评论
- 经验分享/教程

### 🟢 短篇内容 (50-280 字)

**每日目标**: 6 个
**创作时间**: 3-5 分钟/个
**核心价值**: 快速传播、互动引发、观点表达

**适合主题**:
- 新闻/产品发布速评
- 个人感悟/思考碎片
- 转评有价值的内容
- 提问/讨论引发

## 二、渠道写作规范

### Twitter (推文/推文串)

**发布方式**: `post-to-x` skill

#### 单条推文
- **字数**: 不超过 280 字符（中文约 140 字）
- **结构**: 观点/洞察 + 简要论据 + （可选）链接
- **风格**: 中英文混合，专有名词英文
- **emoji**: 适度使用，不堆砌
- **避免**: "你知道吗？"、"线程来了！" 等套路开头
- **好的开头**: 直接抛观点 "Claude Code 最被低估的能力不是写代码，而是..."

#### 推文串 (Thread)
- **长度**: 3-8 条
- **第 1 条**: Hook — 核心观点或反直觉结论，吸引展开阅读
- **中间条**: 每条一个论点 + 支撑证据
- **最后一条**: 总结 + 行动建议 + 互动引导
- **编号**: 使用 1/N 格式

### 个人博客 (ginonotes.com)

**保存位置**: `contents/blog-posts/YYYY-MM-DD/{slug}.md`

**Markdown 格式**:
```markdown
---
title: "标题"
date: YYYY-MM-DD
tags: [tag1, tag2, tag3]
---

## 引言（2-3 句，建立读者共鸣）

## 正文（2-4 个小节）

### 小节标题
内容...

## 我的思考

[个人视角、经验关联、实践建议]

## 小结

[核心要点回顾 + 行动号召]
```

**写作规范**:
- 标题清晰直接，不用标题党
- 引言从具体场景或问题出发
- 论点 + 论据 + 例证结构
- 每段不超过 4 句
- 代码用代码块 + 语言标注

### 微信公众号

**发布方式**: `post-to-wechat` skill
**字数**: 1500-4000 字

**与博客的差异**:
- 段落更短（每段 2-3 句），适配手机阅读
- 更多使用**加粗**标注关键信息
- 小标题可用疑问句，更吸引人
- 结尾加引导关注/转发
- 可添加"延伸阅读"链接列表
- 图片更多（每 500 字至少 1 张）

### 小红书

**字数**: 300-800 字
**发布方式**: 手动复制

**格式**:
```
标题（含 emoji，吸引点击）

开头 1-2 句抓注意力

🔑 关键点 1: ...
🔑 关键点 2: ...
🔑 关键点 3: ...

💡 个人心得/实操建议

#标签1 #标签2 #标签3 #标签4 #标签5
```

**风格**: 简洁实用、列表化、口语但不轻浮
**标签**: 5-10 个，混合热门标签和精准标签

### 即刻

**字数**: 100-500 字
**发布方式**: 手动复制

**风格**: 社区讨论感，像在跟朋友分享发现
**结构**: 观点/发现 + 简要分析 + 邀请讨论
**特点**:
- 可以更随意和口语化
- 鼓励加互动提问
- 适合发表未成型的想法

### 朋友圈

**字数**: 50-200 字
**发布方式**: 手动复制

**风格**: 简短、个人化、生活化
**特点**:
- 最口语化的渠道
- 可以只是一个感悟+配图
- 避免过于专业的术语
- 适合抛出问题引发讨论

### 知乎

**字数**: 2000-5000 字
**发布方式**: 手动复制

**与博客的差异**:
- 更注重论证逻辑和数据引用
- 段间过渡更明确
- 适合用 Q&A 格式展开
- 需要考虑搜索关键词

## 三、一鱼多吃适配规则

### 从长文衍生

| 源内容 | 目标渠道 | 适配方法 |
|--------|---------|---------|
| 博客全文 | 公众号 | 段落拆短 + 加粗关键词 + 加引导语 |
| 博客全文 | 知乎 | 增加论证深度 + Q&A 结构 |
| 博客要点 | 推文串 | 提取 3-5 个核心论点，每条一个 |
| 博客核心观点 | 单条推文 | 最精华的一句话 + 博客链接 |
| 博客结论 | 即刻 | 讨论体 + 互动提问 |
| 博客感悟 | 朋友圈 | 一句感悟 + 配图 |

### 从中篇内容衍生

| 源内容 | 目标渠道 | 适配方法 |
|--------|---------|---------|
| 推文串 | 小红书 | 合并为列表体 + 加标签 |
| 推文串 | 即刻 | 合并为讨论体 + 加提问 |
| 小红书笔记 | 即刻 | 去标签 + 讨论感 |
| 小红书笔记 | 朋友圈 | 提取核心观点，极简化 |

### 从短篇内容衍生

| 源内容 | 目标渠道 | 适配方法 |
|--------|---------|---------|
| 推文 | 即刻 | 加上下文 + 互动提问 |
| 推文 | 朋友圈 | 口语化 + 更个人化 |

## 五、内容质量检查清单

每个内容发布前对照检查：

- [ ] 有个人视角或真实经验（非纯 AI 生成感）
- [ ] 核心观点清晰，一句话能概括
- [ ] 信息准确，数据/引用可追溯
- [ ] 适配目标渠道的风格和字数要求
- [ ] 无错别字、语法错误
- [ ] 术语使用一致（同一概念不用多种称呼）
- [ ] 不与近期已发布内容重复
