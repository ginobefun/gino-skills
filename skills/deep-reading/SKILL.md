# 深度阅读分析 (Deep Reading Analyst)

通过结构化思维框架，将表层阅读转化为深度学习。支持文章链接或直接输入内容。

## When to Use

- 当用户想深度理解一篇文章、论文、报告或长文
- 当用户想拆解论证结构、识别漏洞、提炼洞察或应用特定思维框架
- 当快速摘要不够，需要系统化分析输出时

## When Not to Use

- 需要逐篇引导阅读、记录进度和反思：用 `guide-reading`
- 需要把素材改写成发布内容：用 `synthesize-content`
- 只想知道今天该读什么：用 `curate-daily-content`

## Gotchas

- 默认 Level 3 很重；如果用户只是想快速掌握要点，不要无条件上最深分析
- 输出默认使用中文，这适合大部分场景，但跨语言引用时要保留关键原文术语
- Level 4 才需要跨源研究；不要把普通深读任务自动升级成检索型研究任务
- 若工作区已有缓存正文，应优先复用，避免重复抓取同一篇内容

## Related Skills

- `guide-reading`：管理多篇材料的阅读节奏
- `synthesize-content`：把分析结果转换成可发布内容
- `manage-daily-content`：在更大的日常内容编排中调度深读

## Boundary

本 skill 负责单篇材料的深度分析，不负责：
- 多篇阅读流程管理
- 选题优先级判断
- 正式平台文案生成

## Runtime Conventions

This skill follows `docs/skill-runtime-conventions.md`.

- Reusable state or memory belongs under `${CLAUDE_PLUGIN_DATA}/gino-skills/read-deeply/` or `.gino-skills/data/read-deeply/`
- `contents/tmp/workspace/YYYY-MM-DD/` is a disposable cache shared with sibling skills
- Article fetch cache may live in workspace for same-day reuse, but durable notes or history should move to the stable state root
- If setup is added later, prefer `.gino-skills/read-deeply/config.json`

## 内容获取

### Daily Workspace 缓存集成

获取文章内容前，优先检查 Daily Workspace 缓存（避免重复拉取）：

1. **检查缓存**: 读取 `contents/tmp/workspace/YYYY-MM-DD/article-details/` 目录，按 URL 或文章 ID 匹配
   - 若命中且包含全文内容 → 直接使用缓存，跳过网络获取
2. **未命中时获取**: 通过 Jina Reader 将网页转为 Markdown：
   - 使用 WebFetch 工具访问 `https://r.jina.ai/{用户提供的完整URL}`
   - prompt 设为：`提取文章的完整内容，保留标题、正文、作者、日期等所有信息`
   - 如果 Jina Reader 返回错误、内容为空或被拦截，直接用 WebFetch 访问原始 URL 兜底
3. **写入缓存**: 获取成功后，将全文内容保存到工作区缓存：
   ```bash
   mkdir -p contents/tmp/workspace/YYYY-MM-DD/article-details
   ```
   保存为 `contents/tmp/workspace/YYYY-MM-DD/article-details/{article-id}.md`，格式包含元数据 + 全文内容。
4. **追加分析结果**: 分析完成后，将 AI 分析结果追加到同一缓存文件的 `## AI 分析` 部分，供 content-synthesizer 等下游 skill 直接引用。

文件格式详见 `skills/daily-content-management/references/workspace-spec.md`。

获取后立即进入分析流程，无需向用户确认。

## 输出语言

分析输出**始终使用中文**，无论原文是什么语言。对于英文原文中的专有术语，首次出现时标注英文原文（如"第一性原理（First Principles）"），后续使用中文即可。

## 分析级别

| 级别 | 名称 | 框架组合 | 适用场景 |
|------|------|----------|----------|
| Level 1 | 快速 | SCQA + 5W2H + MECE + 思维导图 | 快速抓住要点 |
| Level 2 | 标准 | Level 1 + 批判性思维 + 逆向思维 | 日常学习理解 |
| **Level 3** | **深度（默认）** | **Level 2 + 心智模型 + 第一性原理 + 系统思维 + 六顶帽 + 哲学/历史视角** | **重要内容深入理解** |
| Level 4 | 研究 | Level 3 + 跨源对比（含网络搜索） | 学术研究、多源综合 |

**默认 Level 3**。仅当用户明确指定其他级别时才调整。不要询问用户选择级别——直接开始分析。

## 框架按需加载

根据分析级别加载对应的参考文件：

- **Level 1+**: 加载 `references/structural_analysis.md`（SCQA + 5W2H + MECE）
- **Level 2+**: 加载 `references/critical_analysis.md`（批判性思维 + 逆向思维）
- **Level 3+**: 加载 `references/deep_analysis.md`（心智模型 + 第一性原理 + 系统思维 + 六顶帽 + 哲学/历史视角）
- **Level 4**: 加载 `references/research_analysis.md`（跨源对比矩阵）

Level 3 默认需加载前三个参考文件。

**所有级别**都输出思维导图。综合输出阶段包含费曼检验、苏格拉底追问和元认知反思。

## 分析工作流

### 阶段一：结构理解（所有级别）

加载 `references/structural_analysis.md`，执行：

**1. 文章概览**

```markdown
## 文章概览

- 类型：[文章/论文/报告/指南/观点文]
- 核心论点：[一句话概括]
- 关键概念：[3-5 个核心术语及简要定义]
```

**2. SCQA 解构**

```markdown
## SCQA 结构

**S（背景）**: [文章建立的背景/现状]
**C（冲突）**: [识别的问题/挑战]
**Q（问题）**: [要解决的核心问题]
**A（答案）**: [主要解决方案/结论]

逻辑完整性：S→C [自然/牵强] | C→Q [直接/间接] | Q→A [充分/不足]
```

**3. MECE 完整性检查**

检查文章的分类和论证是否满足"相互独立、完全穷尽"原则：

```markdown
## MECE 分析

### 分类检查
- 文章的核心分类/分组：[列出]
- 互斥性：[各类别之间是否有重叠？指出重叠之处]
- 穷尽性：[是否覆盖了所有情况？指出遗漏类别]

### 论证覆盖度
- 已覆盖：[文章完整讨论的方面]
- 遗漏：[应该讨论但未涉及的方面]
- 重叠：[重复论述的部分]
```

**4. 5W2H 完整性检查**

```markdown
## 信息完整性

✅ 充分覆盖：[列出已回答的维度]
⚠️ 部分覆盖：[列出回答不完整的维度]
❌ 缺失：[列出完全未涉及的维度]

关键信息缺口：[1-2 个最重要的缺失信息]
```

**5. 思维导图**

将文章的论证结构和概念关系可视化。思维导图是每次分析的标配输出，应捕捉完整论证骨架，而非只是标题层级：

```markdown
## 思维导图

[文章核心主题]
├─ [主论点 1]
│   ├─ [子论点 1.1]
│   │   ├─ [证据/例证]
│   │   └─ [证据/例证]
│   └─ [子论点 1.2]
├─ [主论点 2]
│   ├─ [子论点 2.1]
│   └─ [子论点 2.2]
├─ [主论点 3]
│   └─ ...
└─ [结论/行动建议]
    ├─ [行动 1]
    └─ [行动 2]

概念关联（横向联系）:
- [概念 A] ←→ [概念 B]: [互相支撑/矛盾]
- [概念 C] → [概念 D]: [因果/依赖]
- [概念 E] ←→ [外部概念 F]: [跨域关联]
```

### 阶段二：批判性分析（Level 2+）

加载 `references/critical_analysis.md`，执行：

**1. 论证质量评估**

```markdown
## 批判性分析

### 论证强度：[X/10]

**优势：**
- [有力论点及其证据]

**弱点：**
- [逻辑缺陷或证据不足之处]

**逻辑谬误检测：**
- [如有，列出具体谬误类型及出处]

**证据评估：**
| 证据 | 类型 | 可信度 | 备注 |
|------|------|--------|------|
| [证据 1] | [数据/案例/专家观点] | [高/中/低] | [评价] |
```

**2. 逆向思维分析**

```markdown
## 逆向分析

### 如何确保失败？
1. [失败模式 1] → 预防：[...]
2. [失败模式 2] → 预防：[...]

### 假设反转测试
- 假设 1: [核心假设] → 若不成立：[后果]
- 假设 2: [核心假设] → 若不成立：[后果]

### 被忽略的风险
- [未提及的风险因素]
```

### 阶段三：深度分析（Level 3，默认）

加载 `references/deep_analysis.md`，执行：

**1. 心智模型多维分析**

从不同学科选择 3-5 个相关模型，应用于文章内容：

```markdown
## 多模型分析

### 1. [模型名称]（[学科]）
洞察：[该视角下的发现]

### 2. [模型名称]（[学科]）
洞察：[该视角下的发现]

### 3. [模型名称]（[学科]）
洞察：[该视角下的发现]

跨模型交叉洞察：[多个模型指向同一结论的发现]
```

**2. 第一性原理拆解**

```markdown
## 第一性原理

### 隐含假设
1. [假设 1] → 是否根本成立：[是/否/有条件]
2. [假设 2] → 是否根本成立：[是/否/有条件]

### 基本事实
[剥离假设后，可验证的根本事实]

### 重建理解
从基本事实出发重新推导 → [重建后的结论与原文对比]
```

**3. 系统思维映射**

```markdown
## 系统分析

### 因果关系图
[变量 A] ──增强──> [变量 B]
      ↑                    |
      |                    |
   平衡               增强
      |                    |
      └────────<───────────┘

### 反馈回路
- 增强回路：[识别的正反馈循环]
- 平衡回路：[识别的负反馈循环]

### 杠杆点
[最小投入产生最大影响的关键干预点]
```

**4. 六顶帽全方位评估**

```markdown
## 六顶帽视角

🤍 事实：[客观数据和已知信息]
❤️ 直觉：[直觉反应和情感共鸣]
🖤 风险：[潜在问题和不利因素]
💛 价值：[积极面和机会]
💚 创意：[延伸想法和替代方案]
💙 总结：[综合判断和元思考]
```

**5. 哲学与历史视角**

从人文学科角度深化对文章的理解：

```markdown
## 哲学视角

### 认识论（知识如何成立）
- 文章的知识主张基于什么？（经验/理性/权威/共识）
- 知识的确定性程度：[确定/高度可能/推测性]

### 伦理学（道德含义）
- 文章方案的伦理影响：[谁受益？谁可能受损？]
- 隐含的价值取向：[效率优先/公平优先/自由优先/...]

### 本体论（事物本质）
- 文章对核心事物本质的假设：[如何定义"成功"/"价值"/"好的"？]

## 历史视角

### 历史语境
- 该议题的历史脉络：[过去如何演变到今天？]
- 历史上的类似情境：[有无可比案例？结果如何？]

### 历史规律
- 是否符合某种周期性模式？
- 历史教训：[过去的经验对当前议题有何启示？]
```

### 阶段四：跨源研究（Level 4）

加载 `references/research_analysis.md`。

使用 WebSearch 搜索 2-3 篇相关文章，用 Jina Reader 获取内容，然后执行跨源对比分析。

```markdown
## 多源对比

### 来源 1: [本文]
核心主张：[...] | 证据类型：[...] | 关键假设：[...]

### 来源 2: [搜索到的文章]
核心主张：[...] | 证据类型：[...] | 关键假设：[...]

### 来源 3: [搜索到的文章]
核心主张：[...] | 证据类型：[...] | 关键假设：[...]

### 综合
- **共识**: [所有来源一致的观点]
- **分歧**: [观点差异及原因分析]
- **独特贡献**: [每个来源的独特价值]
- **整合观点**: [综合多源后的完整理解]
```

### 阶段五：综合输出

将分析转化为对读者真正有用的知识。各子模块按级别逐步加入：

- **5.1 核心洞察** — 所有级别
- **5.2 费曼检验** — Level 2+
- **5.3 苏格拉底追问** — Level 3+
- **5.4 元认知反思与个人建议** — Level 3+
- **5.5 收尾** — 所有级别

#### 5.1 核心洞察（所有级别）

```markdown
## 核心洞察（Top 3）

### 1. [洞察标题]
**内容**: [用自己的话重述]
**为什么重要**: [与读者的关联]
**一个行动**: [具体、可执行的下一步]

### 2. [洞察标题]
（同上格式）

### 3. [洞察标题]
（同上格式）
```

#### 5.2 费曼检验（Level 2+）

用最简单的语言重述文章核心观点，暴露理解盲区：

```markdown
## 费曼检验：用大白话说

**一句话版本**: [不用任何术语，像对朋友解释一样]

**类比**: [核心观点] 就像 [日常生活中的类比]，因为 [相似之处]

**常见误解**:
- 人们容易以为：[误解]
- 实际上：[纠正]

**理解盲区**:
在解释过程中，发现以下概念仍需深入理解：
- [盲区 1]
- [盲区 2]
```

#### 5.3 苏格拉底式追问（Level 3+）

通过层层追问，触达文章未及的深层问题：

```markdown
## 苏格拉底追问

### 定义追问
- 文章中 [核心概念] 的定义是否清晰？是否有其他合理定义？

### 假设追问
- 这个论证依赖什么未声明的前提？

### 证据追问
- 我们怎么知道这是对的？证据链是否完整？

### 视角追问
- 如果从 [对立方] 的角度看，这个问题会怎样？

### 推论追问
- 如果文章的结论成立，逻辑上还会导致什么？有没有作者未意识到的推论？

### 元问题
- 这个问题本身是不是正确的问题？有没有更好的问题应该被提出？
```

#### 5.4 元认知反思与个人认知建议（Level 3+）

引导读者审视自己的阅读过程，并结合对用户的了解输出个性化建议。

查阅 auto memory 目录中的用户信息（工作背景、关注领域、知识体系），用于生成个性化建议。如果没有足够背景信息，给出通用建议并提示用户可以补充个人背景以获得更精准的建议。

```markdown
## 元认知反思

### 阅读过程审视
- 本文中最可能让读者困惑的部分：[指出并解释]
- 本文挑战的常见认知：[原有认知 → 文章提出的新观点]
- 本文可能引发的情绪反应：[指出哪些内容可能引发认同/抵触，以及为什么]

### 认知偏差提醒
- 确认偏差风险：[读者是否可能因已有立场而过度认同/抵触本文？]
- 光环效应：[是否因作者/来源权威而降低了批判性？]
- 框架效应：[文章的叙述方式是否引导了特定结论？]

### 对你的认知和行动建议

**认知层面**:
- [这篇文章如何补充/挑战/扩展你已有的知识体系]
- [建议重新审视的已有认知]
- [与你此前阅读/关注的内容的连接点]

**行动层面**:
- [基于你的具体场景，最值得尝试的一件事]
- [建议纳入工作/学习流程的改变]

**知识管理建议**:
- [建议关联的已有知识]
- [建议深入探索的方向]
- [建议与谁讨论以获得新视角]
```

#### 5.5 收尾（所有级别）

```markdown
## 立即行动

[一件 24 小时内可以尝试的具体小事]

## 延伸方向

- 进一步阅读：[相关推荐]
- 实践验证：[如何在真实场景中测试]
- 知识连接：[与哪些已知概念关联]

## 使用的思维框架
✅ SCQA ✅ 5W2H ✅ MECE ✅ 思维导图 [根据实际使用情况标注 ✅ 或 ☐]
```

## 内容类型自动适配

根据内容类型，自动调整分析侧重：

| 内容类型 | 重点框架 | 特别适合的新增维度 |
|----------|----------|--------------------|
| 商业/战略文章 | SCQA + MECE + 心智模型 + 逆向思维 | 历史视角（商业周期）|
| 学术论文/研究 | 5W2H + 批判性思维 + 系统思维 | 认识论（知识主张基础）|
| 技术教程/指南 | SCQA + 5W2H + 第一性原理 | 费曼检验（测试理解深度）|
| 观点/评论文 | 批判性思维 + 逆向思维 + 六顶帽 | 哲学视角（伦理与价值取向）|
| 案例分析 | SCQA + MECE + 心智模型 + 系统思维 | 历史视角（类似案例对比）|
| 趋势预测/未来展望 | 系统思维 + 逆向思维 + 心智模型 | 历史视角（周期规律）|
| 个人成长/方法论 | SCQA + 第一性原理 + 六顶帽 | 费曼检验 + 苏格拉底追问 |

这是建议性的侧重调整，不影响框架的完整应用。所有级别对应的框架都应执行。

## 质量标准

每次分析必须：
- 忠于原文内容，不歪曲作者观点
- 区分事实与观点
- 提供具体例证而非泛泛而谈
- 恰当使用框架（不生搬硬套）
- 引用原文具体段落或论述
- 以可执行的行动步骤收尾

避免：
- 一次堆砌所有框架（尊重级别设定）
- 学术术语不做解释
- 只分析不给行动建议
- 逐字复制原文（用自己的语言重述）
- 表面化地套用框架（深入而非广泛）

## 用户交互适配

当用户在对话中追问时，灵活响应：
