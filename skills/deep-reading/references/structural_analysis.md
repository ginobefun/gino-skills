# 结构化分析框架：SCQA + 5W2H + MECE

## 一、SCQA 框架（麦肯锡结构化思维）

### 核心概念

SCQA 将复杂问题拆解为四个叙事要素，揭示文章的论证骨架：

- **S（Situation/背景）**: 稳定的、双方认可的事实背景
- **C（Complication/冲突）**: 打破平衡的变化、问题或挑战
- **Q（Question/问题）**: 由冲突引发的核心问题
- **A（Answer/答案）**: 解决问题的方案或结论

### 识别方法

**识别 S（背景）**：
- 寻找行业现状、历史发展、市场环境等客观描述
- 特征：陈述事实、无主观判断、为后续冲突铺垫
- 提取公式："在[时间]，[主体]在[领域]处于[状态]"

**识别 C（冲突）**：
- 寻找转折信号词：但是、然而、问题在于、挑战是、令人担忧的是
- 类型：外部变化（市场/技术/政策）、内部问题（资源/能力）、期望差距、新兴威胁
- 好的 C 能制造紧迫感，说明"为什么需要改变"

**识别 Q（问题）**：
- 显式问题：文章直接提出的问号
- 隐式问题：从 S→C 逻辑推导出的、作者想回答但未明说的问题
- 问题类型：决策型（A还是B？）、方法型（如何实现X？）、原因型（为什么Y？）、预测型（会怎样？）

**识别 A（答案）**：
- 寻找核心建议、解决方案、推导结论、行动计划
- 结构类型：单一方案、多步骤方案、框架工具、指导原则

### 评估维度

| 评估项 | 评分标准 |
|--------|----------|
| S 客观性 | 是否为事实陈述而非主观判断 |
| C 紧迫性 | 是否制造了真实的张力 |
| Q 精准性 | 是否从 C 直接衍生且可回答 |
| A 可行性 | 是否直接回应 Q 且具备操作性 |
| 逻辑连贯 | S→C→Q→A 每步过渡是否自然 |

### SCQA 变体

- **标准型 S-C-Q-A**：问题导向文章
- **开门见山 A-S-C-Q-A**：先抛结论，适合执行摘要
- **悬念型 C-S-Q-A**：先抛冲突抓眼球
- **层进型 S-C-Q-A-Q-A**：多层递进分析

### 文章缺乏明显 SCQA 时的处理

即使文章结构混乱，也可以主动重建：
1. 提取信息碎片
2. 归类到 SCQA 四个桶
3. 识别缺失的部分
4. 重组理解

## 二、5W2H 完整性分析

### 核心概念

通过 7 个基本问题系统检查信息完整性，发现遗漏和盲区：

| 维度 | 核心问题 | 深层追问 |
|------|----------|----------|
| **What（内容）** | 主题是什么？核心主张？ | 真正的问题是什么？遗漏了什么？ |
| **Why（原因）** | 为什么讨论这个？为什么有效？ | 为什么不选其他方案？（五个为什么追问法） |
| **Who（人物）** | 目标受众？利益相关者？ | 谁受益？谁受损？作者有何偏见？ |
| **When（时间）** | 何时开始？时间表？ | 时间期望是否合理？关键里程碑？ |
| **Where（场景）** | 适用场景？限制条件？ | 换个环境还有效吗？规模影响？ |
| **How（方法）** | 具体步骤？使用什么工具？ | 如何衡量进展？如何处理障碍？ |
| **How much（成本）** | 投入多少？预期回报？ | ROI 如何？风险多大？ |

### 五个为什么追问法（Why 维度深化）

对表面问题连续追问 5 次"为什么"，触达根本原因：

```
问题: [表面现象]
→ 为什么 1: [第一层]
  → 为什么 2: [第二层]
    → 为什么 3: [第三层]
      → 为什么 4: [第四层]
        → 为什么 5: [根本原因] ← 真正问题
```

### 按内容类型调整权重

不是所有 7 个问题同等重要，按文章类型调整：

- **战略文章**: Why (40%) > What (30%) > How (20%) > 其他
- **操作指南**: How (40%) > What (25%) > How much (20%) > 其他
- **分析报告**: What (30%) > Why (30%) > Who (20%) > 其他

### 完整性评估输出

```
充分覆盖 (✅): [列出 7 维度中回答充分的]
部分覆盖 (⚠️): [回答不完整的，注明缺什么]
完全缺失 (❌): [未涉及的维度]

信息完整度: [X/7]
可行性评分: [基于完整度的 1-10 分]
关键补充建议: [需要补充的最重要信息]
```

### 与 SCQA 的配合

SCQA 理解整体框架，5W2H 检查细节完整性：
- S 的背景是否充分？（When, Where）
- C 的问题是否清晰？（What, Why）
- A 的方案是否可行？（How, How much, Who）

## 三、MECE 分析（相互独立、完全穷尽）

### 核心概念

MECE（Mutually Exclusive, Collectively Exhaustive）是麦肯锡的核心分析原则：
- **互斥（ME）**: 每个类别之间不重叠，同一事物不会同时属于两个类别
- **穷尽（CE）**: 所有类别合在一起覆盖了全部可能性，没有遗漏

### 分析文章时的应用

#### 1. 检查文章的分类体系

文章在论述中常进行分类（如"三种策略""四个阶段"），MECE 检查这些分类是否严谨：

```
分类: [文章提出的分类]
互斥性检查:
- 类别 A 和 B 是否有重叠？→ [是/否，说明]
- 类别 B 和 C 是否有重叠？→ [是/否，说明]
穷尽性检查:
- 是否存在不属于任何类别的情况？→ [列出遗漏]
MECE 评级: [严格MECE / 基本MECE / 有重叠 / 有遗漏 / 既重叠又遗漏]
```

#### 2. 检查论证覆盖度

用 MECE 原则审视文章是否全面讨论了问题的各个方面：

**常用 MECE 切分维度**：
- 内部 vs 外部因素
- 短期 vs 中期 vs 长期
- 战略层 vs 战术层 vs 执行层
- 供给侧 vs 需求侧
- 定量 vs 定性
- 已知 vs 未知
- 可控 vs 不可控

选择最适合文章主题的切分维度，检查每个维度下文章是否有充分讨论。

#### 3. 与 SCQA/5W2H 的协同

| 框架 | 检查什么 | MECE 的补充 |
|------|----------|-------------|
| SCQA | 论证骨架是否完整 | A（答案）的方案分类是否 MECE |
| 5W2H | 7 个维度是否覆盖 | 每个维度下的讨论是否 MECE |

### 常见 MECE 违规

- **伪穷尽**: "主要有三种…" 但实际还有重要的第四种
- **伪互斥**: 类别之间界限模糊，同一案例可归入多类
- **偷换粒度**: 分类层级不一致（如把战略级和操作级混在一起）
- **遗漏"其他"**: 用"其他"兜底不算真正穷尽，需要问"其他里有什么重要的？"
